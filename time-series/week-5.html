2 Recap: The ETS Family

All three classical exponential smoothing methods belong to the ETS (Error, Trend, Seasonal) family. They differ only in which components they model:

Method Level (α) Trend (β) Seasonality (γ)
SES ✅ ❌ ❌
Holt ✅ ✅ ❌
Holt-Winters ✅ ✅ ✅

💡 The more complex the data (trend, seasonality), the more parameters you need.

3 Core Explanations

3.1 Single Exponential Smoothing (SES) – Quick Recap

Use when: Your data has no trend and no seasonality — just a fluctuating level.

The key idea is that the prediction equals the level, which is a weighted average of past observations. The weight of each past observation decays exponentially as you go further back in time.

  • α close to 1 → short memory → only recent observations matter
  • α close to 0 → long memory → many past observations are used

The forecast is constant: every future prediction is the same value, only the confidence interval grows wider the further out you predict.

✏️ Your annotation space:

3.2 Holt’s Method (Double Exponential Smoothing)

Use when: Your data has a trend (upward or downward), but no seasonality.

Holt’s method adds a trend component on top of the level.

3.2.1 How it works

The forecast equation is similar to the equation of a straight line:

\[\hat{y}_{t+h} = \ell_t + h \cdot b_t\]

Where:

  • \(\ell_t\) = level at time \(t\) (how far back to look for relevant information)
  • \(b_t\) = trend / slope at time \(t\)
  • \(h\) = number of steps ahead

The level is updated using α (0 to 1) and the trend is updated using β (0 to 1), both found by minimising the Sum of Squared Errors (SSE) — R does this automatically using quadratic programming.

💡 Think of it like fitting a line of best fit, but the line is constantly being updated as new data arrives.

✏️ Your annotation space:

3.3 Damped Trend (Holt with Damping)

Use when: The trend is not linear — it may slow down, flatten, or follow a cyclical pattern over time.

The damped trend adds a φ (phi) parameter that “bends” the trend forecast, preventing it from projecting indefinitely upward or downward. This is useful when the data contains a cyclical component (long-term oscillation, not to be confused with short-term seasonality).

  • A polynomial trend with more degrees captures more complex shapes.
  • The lecture referred to this as the damp component in R output.

✏️ Your annotation space:

3.4 Holt-Winters Method (Triple Exponential Smoothing)

Use when: Your data has trend AND seasonality.

Three parameters are estimated:

  • α → level
  • β → trend
  • γ (gamma) → seasonality

3.4.1 Additive vs. Multiplicative Seasonality

Type When to use Behaviour
Additive Seasonal swings are constant in size Pattern has the same amplitude over time
Multiplicative Seasonal swings grow with the level Pattern amplitude increases as data increases

💡 If your time series looks like a trumpet (widening fluctuations), use multiplicative. If the ups and downs stay the same height, use additive.

The lecture noted that multiplicative better captures a stable seasonal pattern, while additive can misrepresent a growing seasonal effect.

✏️ Your annotation space:

4 Key Equations / Formulas

4.1 SES Forecast

\[\hat{y}_{t+1} = \ell_t = \alpha y_t + (1 - \alpha)\ell_{t-1}\]

4.2 Holt’s Method Forecast

\[\hat{y}_{t+h} = \ell_t + h \cdot b_t\]

\[\ell_t = \alpha y_t + (1 - \alpha)(\ell_{t-1} + b_{t-1})\]

\[b_t = \beta(\ell_t - \ell_{t-1}) + (1 - \beta)b_{t-1}\]

4.3 Confidence Interval (general form)

\[\hat{y} \pm z^* \cdot \frac{\sigma}{\sqrt{n}}\]

Where \(z^* = 1.96\) for 95% confidence (standard normal distribution).

4.4 Error Metrics

Metric Formula Key Property
MAE \(\frac{1}{n}\sum \|y_t - \hat{y}_t\|\) Average absolute error
MSE \(\frac{1}{n}\sum (y_t - \hat{y}_t)^2\) Penalises large errors more
MAPE \(\frac{1}{n}\sum \left|\frac{y_t - \hat{y}_t}{y_t}\right| \times 100\) Unit-free — good for comparing different scales

💡 Prefer MSE when large errors are costly. Use MAPE when comparing models across different datasets or units.

5 R Code Examples

5.1 Fitting models with the forecast library

library(forecast)

# Single Exponential Smoothing
fit_ses <- ses(data, h = 5)

# Holt's method (trend, no seasonality)
fit_holt <- holt(data, h = 5)

# Holt with damped trend
fit_holt_damp <- holt(data, damped = TRUE, h = 5)

# Holt-Winters (additive seasonality)
fit_hw_add <- hw(data, seasonal = "additive", h = 12)

# Holt-Winters (multiplicative seasonality)
fit_hw_mult <- hw(data, seasonal = "multiplicative", h = 12)

# ETS automatic (R chooses best model minimising BIC)
fit_ets <- ets(data)
summary(fit_ets)

# ETS with specified components (additive error, additive trend, additive seasonal)
fit_ets_aaa <- ets(data, model = "AAA")
summary(fit_ets_aaa)

5.2 Comparing models using error metrics

# Extract accuracy metrics
accuracy(fit_ses)
accuracy(fit_holt)
accuracy(fit_holt_damp)

# Compare AIC / BIC (lower is better)
fit_ses$model$aic
fit_holt$model$aic

5.3 Plotting forecasts

plot(forecast(fit_holt, h = 10))

Model MSE MAE

  • Data: number of hotel stays (millions)
  • Used to compare Holt-Winters additive vs. multiplicative
  • Multiplicative better captured the stable, consistent seasonal pattern
  • Additive produced a non-constant (incorrect) seasonal component

7 Information Criteria: AIC and BIC

AIC (Akaike Information Criterion) and BIC (Bayesian Information Criterion) are used to compare models:

  • Values have no meaning on their own
  • Only useful for comparing two models on the same dataset
  • Lower value = better model

10 Summary Points

  • SES = level only → flat forecast, growing confidence interval
  • Holt = level + trend → forecast follows a line
  • Holt damped = level + non-linear/cyclical trend → curved forecast that stabilises
  • Holt-Winters = level + trend + seasonality → use additive if variance is constant, multiplicative if it grows
  • All methods belong to the ETS family and can be called using ets() in R
  • To compare models: use MSE or MAE for residual quality; AIC/BIC for model selection
  • MAPE is preferred when comparing across datasets with different units or scales
  • Always analyse residuals: they should be normally distributed, centred at zero, with constant variance and be independent
  • R minimises SSE automatically — you do not need to do this by hand
