A living reference for lecture content, lab work, and exam prep. Browse by module in the sidebar, or follow the guide below to run this site locally or push updates to GitHub Pages.

  • Lab practice & R examples
  • Revision summaries
  • Exam preparation notes

Running this site locally

Ideal when you want to preview changes before publishing — works offline too.

1

Open your terminal and navigate to the project folder. If you cloned the repo, it'll be wherever you saved it.

cd data-analytics-notes
2

Start the Quarto preview server. This builds the site and opens it in your browser automatically.

quarto preview

The site will live-reload whenever you save a .qmd file — no need to restart.

3

Open your browser and go to the address shown in the terminal — usually:

http://localhost:4321
✓ Tip: Press Ctrl + C in the terminal when you're done to stop the preview server.

Publishing to GitHub Pages

Quarto has a built-in command for this — no manual file copying needed.

1

Make sure GitHub Pages is set up. In your repo on GitHub, go to Settings → Pages and set the source to the gh-pages branch. You only need to do this once.

2

Commit your changes to the main branch as normal. Quarto publishes separately from your source files.

git add . git commit -m "Update week 3 notes" git push
3

Publish with Quarto. This builds the site and pushes the rendered output to your gh-pages branch in one go.

quarto publish gh-pages

The first time you run this, Quarto may ask you to confirm the publishing destination — just hit Enter.

4

Wait ~30–60 seconds, then visit your site. GitHub Pages takes a moment to deploy the new build.

https://lgolin.github.io/data-analytics-notes/
⚠ Note: Don't edit the gh-pages branch directly — it's managed by Quarto. Always make edits in your source files and re-run quarto publish gh-pages.
