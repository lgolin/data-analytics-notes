Time Series – Week 7: Classical Models Analysis (Step-by-Step Template)

STAT9005 – Study Guide

📝 Note for reviewer: These notes are based on the Week 7 lecture transcript. Leave comments or annotations in the sections marked with <!-- ADD MORE HERE -->.

1 Key Topics Covered This Week

  • Step-by-step template for analysing a time series (Classical Models)
  • How to identify components: trend and seasonality
  • Additive vs. Multiplicative decomposition
  • How to analyse and standardise residuals
  • Fitting Exponential Smoothing models: SES, Holt, Holt-Winters
  • Comparing models using goodness-of-fit metrics (MSE, AIC, BIC)
  • Practical walkthroughs: Air Passengers, Wine Industry, Air Miles, Unemployment, US Accidents

2 Core Explanations

2.1 The Analysis Template

The main goal of this week was to learn a reusable template — a series of steps you can follow for almost any univariate time series dataset. The steps are:

  1. Plot the data – Look at the time series visually.
  2. Identify components – Does it have trend? Seasonality? Both?
  3. Decompose the series – Separate it into its parts.
  4. Analyse the residuals – Check if the “leftover” noise looks random and normal.
  5. Fit models – Try different Exponential Smoothing models.
  6. Compare models – Use error metrics and information criteria to choose the best.
  7. Check final residuals – Make sure the chosen model’s errors are well-behaved.
TipWhy use a template?

A template helps you stay consistent and not miss important steps. Most time series follow the same kind of analysis, so this checklist works for the majority of datasets.

2.2 Step 1 – Plot and Identify Components

The first thing you do is plot your data. Ask yourself:

  • Is there a trend? (Does the data go up or down over time?)
  • Is there seasonality? (Does the pattern repeat at regular intervals, like every 12 months?)
  • Is the seasonality additive (same size each year) or multiplicative (getting bigger each year)?

Example – Air Passengers dataset (1949–1960):

  • Clear increasing trend over time.
  • Seasonality present – repeating every 12 months (monthly data → periodicity = 12).
  • The seasonal pattern gets bigger over time → this suggests multiplicative seasonality.
# Load the dataset
data("AirPassengers")

# Plot the time series
autoplot(AirPassengers)

2.3 Step 2 – Decompose the Time Series

Decomposition means separating the time series into its parts:

  • Trend component
  • Seasonal component
  • Residual (random) component

You can do this two ways:

  • Additive decomposition: assumes seasonal effect is constant → \(Y_t = T_t + S_t + R_t\)
  • Multiplicative decomposition: assumes seasonal effect grows with the trend → \(Y_t = T_t \times S_t \times R_t\)
# Additive decomposition
decomposed_add <- decompose(AirPassengers, type = "additive")
plot(decomposed_add)

# Multiplicative decomposition
decomposed_mult <- decompose(AirPassengers, type = "multiplicative")
plot(decomposed_mult)
NoteHow to choose: Additive or Multiplicative?

Look at the residual (random) component:

  • You want residuals that look random, with no pattern, and with constant variance.
  • If the multiplicative residuals look more random → use multiplicative.
  • For Air Passengers: multiplicative was better because the seasonal variation grows over time.

📌 Missing values in decomposition: You will always see 6 missing values at the start and 6 at the end (for monthly data). This is normal — it happens because the moving average needs observations on both sides. Formula: missing values = periodicity / 2 = 12 / 2 = 6

2.4 Step 3 – Analyse the Residuals

After decomposition, you need to check the residuals (the random part). Good residuals should:

  1. Have a mean close to zero
  2. Have constant variance (the spread doesn’t change over time)
  3. Look random (no visible pattern)
  4. Be approximately normally distributed

2.4.1 Standardising the Residuals

To better understand and compare residuals, we standardise them. This means rescaling them so that:

  • Mean = 0
  • Standard deviation = 1

\[z = \frac{x - \bar{x}}{s}\]

Where:

  • \(x\) = your residual value
  • \(\bar{x}\) = mean of all residuals
  • \(s\) = standard deviation of all residuals

After standardising, you can use the ±2 rule: 95% of values in a normal distribution fall between -2 and +2. This makes it easy to spot outliers.

# Extract residuals from multiplicative decomposition
residuals_mult <- decomposed_mult$random

# Standardise the residuals
mean_res <- mean(residuals_mult, na.rm = TRUE)
sd_res   <- sd(residuals_mult, na.rm = TRUE)
std_res  <- (residuals_mult - mean_res) / sd_res

# Plot: histogram, boxplot, QQ plot, scatter
par(mfrow = c(2, 2))
hist(std_res, main = "Histogram of Standardised Residuals", xlab = "")
boxplot(std_res, main = "Boxplot")
qqnorm(std_res); qqline(std_res, col = "red")
plot(std_res, main = "Scatter Plot of Residuals")
abline(h = c(-2, 2), col = "blue", lty = 2)
TipReading the residual plots
Plot What you look for
Histogram Roughly bell-shaped (normal curve)
Boxplot Symmetric around 0, not too many outliers outside ±2
QQ Plot Points close to the red diagonal line (especially between ±2)
Scatter Random scatter, constant width band (no funnel shape)

2.5 Step 4 – Fitting Classical Models

2.5.1 Overview of the Models

The three main classical models are:

Model Also called Components
SES Single Exponential Smoothing Level only (no trend, no seasonality)
Holt Double Exponential Smoothing Level + Trend
Holt-Winters Triple Exponential Smoothing Level + Trend + Seasonality

For datasets with both trend and seasonality (like Air Passengers), you use Holt-Winters.

2.5.2 Using ETS in R (Automatic Model Selection)

The ets() function in R automatically selects the best model:

library(forecast)

# Let R choose the best model automatically
best_model <- ets(AirPassengers)
summary(best_model)

The output tells you:

  • Error type: Additive (A) or Multiplicative (M)
  • Trend type: None (N), Additive (A), Damped (Ad)
  • Seasonal type: None (N), Additive (A), Multiplicative (M)

For Air Passengers, R suggested: M, Ad, M → Multiplicative error, Damped trend, Multiplicative seasonality.

2.5.3 Fitting Holt-Winters Manually

You can also try specific models yourself and compare them:

# Model 1: Additive (expected to be worst for Air Passengers)
hw_add <- hw(AirPassengers, h = 24, seasonal = "additive")

# Model 2: Multiplicative, linear trend
hw_mult_lin <- hw(AirPassengers, h = 24, seasonal = "multiplicative", damped = FALSE)

# Model 3: Multiplicative, damped trend
hw_mult_damp <- hw(AirPassengers, h = 24, seasonal = "multiplicative", damped = TRUE)

# Plot all three for visual comparison
autoplot(hw_add)
autoplot(hw_mult_lin)
autoplot(hw_mult_damp)

2.6 Step 5 – Comparing Models (Goodness of Fit)

Once you have multiple models, you need to decide which is the best one. You use these metrics:

2.6.1 Error Metrics

Metric Full name What it measures
MSE Mean Squared Error Average of squared prediction errors
RMSE Root Mean Squared Error Square root of MSE – same units as data
MAE Mean Absolute Error / Deviation Average of absolute errors (not penalised)
ImportantWhy RMSE vs MAE?

RMSE penalises large errors more than MAE, because it squares the distances first. This means a model with a few large mistakes will have a much higher RMSE than MAE. Prefer RMSE when big errors are especially bad.

2.6.2 Information Criteria

Metric Full name Use
AIC Akaike Information Criterion Balance between fit and complexity
AICc Corrected AIC Better for small samples (< 30 obs)
BIC Bayesian Information Criterion Similar to AIC, penalises complexity more

Lower AIC/BIC = better model. But AIC has a crucial interpretation:

AIC balances model fit and model complexity. If adding one more parameter (making the model more complex) reduces the errors only a little, the AIC will tell you it’s NOT worth the added complexity.

# Compare accuracy metrics for all three models
accuracy(hw_add)
accuracy(hw_mult_lin)
accuracy(hw_mult_damp)

# Compare AIC/BIC
AIC(hw_add$model)
AIC(hw_mult_lin$model)
AIC(hw_mult_damp$model)

2.6.3 Example – Air Passengers Result

Model RMSE AIC
Additive highest highest
Multiplicative, Linear lower lowest ✓
Multiplicative, Damped lowest RMSE slightly higher
  • According to AIC: Model 2 (multiplicative, linear) is best → simpler model, not worth adding the extra damping parameter.
  • According to RMSE: Model 3 (damped) is best → smaller prediction errors.
  • Decision: Prefer the model with the lowest AIC (Model 2), because the improvement in RMSE from adding a 4th parameter is not worth the added complexity.

2.7 Step 6 – Verify Final Model Residuals

Once you pick your best model, you must check its residuals one more time. The same rules apply:

  • Residuals should be normally distributed, mean ≈ 0, constant variance, random pattern.
  • If the residuals look good → you can trust your forecast and confidence intervals.
  • If the residuals are not normal → your confidence intervals may not be reliable.
# Get residuals from chosen model
final_residuals <- residuals(hw_mult_lin)

# Standardise and plot
std_final <- (final_residuals - mean(final_residuals)) / sd(final_residuals)

par(mfrow = c(2, 2))
hist(std_final, main = "Final Model Residuals")
boxplot(std_final)
qqnorm(std_final); qqline(std_final, col = "red")
plot(std_final); abline(h = c(-2, 2), col = "blue", lty = 2)

2.8 Key Equations and Formulas

2.8.1 Standardisation Formula

\[z_t = \frac{r_t - \bar{r}}{s_r}\]

Where \(r_t\) is the residual at time \(t\), \(\bar{r}\) is the mean, and \(s_r\) is the standard deviation.

2.8.2 Additive Decomposition

\[Y_t = T_t + S_t + R_t\]

2.8.3 Multiplicative Decomposition

\[Y_t = T_t \times S_t \times R_t\]

2.8.4 ETS Model Notation

\[\text{ETS}(E, T, S)\]

  • \(E\) = Error type: A (additive) or M (multiplicative)
  • \(T\) = Trend: N (none), A (additive/linear), Ad (additive damped)
  • \(S\) = Seasonality: N (none), A (additive), M (multiplicative)

2.9 Practical Examples from This Class

2.9.1 Wine Industry Dataset

  • Trend: present, possibly cyclical (hidden cycle)
  • Seasonality: present, periodicity = 12
  • Best model: Multiplicative seasonality + Damped trend (confirmed by AIC)
  • Key insight: AIC confirmed it was worth adding the damp parameter here (unlike Air Passengers)

2.9.2 Air Miles Dataset

  • Trend only: no seasonality (annual data, frequency = 1)
  • Models compared: Holt linear vs. Holt damped
  • Issue: Only 24 observations → small sample → residual plots will NOT look perfect. This is expected.
  • Best model: Linear trend model had lower AIC

2.9.3 Unemployment Dataset

  • Negative trend (decreasing) + Seasonality
  • Best model: Holt-Winters (all three components needed)
  • ETS result: Linear trend (no damping needed)

2.9.4 US Accidents Dataset (1973–1978)

  • Seasonality present, periodicity = 12
  • Trend: unclear – could be no trend or a slight curve
  • Best model: Additive seasonality + Damped trend
    • AIC barely higher than the “no trend” version, but RMSE was much better
  • Key lesson: Sometimes you must weigh AIC vs. RMSE and use your judgment
NoteEuro Stock Market (Question 5)

This dataset was not analysed with classical methods this week. It will be covered with ARIMA models starting next week.

3 Summary Points

  • 🔍 Always start by plotting the data and identifying trend and seasonality visually.
  • ➕ or ✖️ Additive vs. Multiplicative: If seasonality grows over time → multiplicative. If constant → additive.
  • 📊 Decompose the series and check the residuals for randomness, normality, and constant variance.
  • 📏 Standardise residuals (mean=0, sd=1) so you can use the ±2 rule for the 95% boundary.
  • 🤖 Use ets() in R as a starting hint, then try other models manually.
  • 📉 AIC balances fit vs. complexity. Adding a parameter is only worth it if the error reduction is large.
  • ✅ After selecting a model, always verify the final residuals – good residuals = trustworthy forecasts.
  • 📝 The exam will ask you to apply this full pipeline to a dataset (open book).

## Key Equations and Formulas

### Standardisation Formula

$$z_t = \frac{r_t - \bar{r}}{s_r}$$

Where $r_t$ is the residual at time $t$, $\bar{r}$ is the mean, and $s_r$ is the standard deviation.

### Additive Decomposition

$$Y_t = T_t + S_t + R_t$$

### Multiplicative Decomposition

$$Y_t = T_t \times S_t \times R_t$$

### ETS Model Notation

$$\text{ETS}(E, T, S)$$

-   $E$ = Error type: **A** (additive) or **M** (multiplicative)
-   $T$ = Trend: **N** (none), **A** (additive/linear), **Ad** (additive damped)
-   $S$ = Seasonality: **N** (none), **A** (additive), **M** (multiplicative)

### Wine Industry Dataset

-   **Trend**: present, possibly cyclical (hidden cycle)
-   **Seasonality**: present, periodicity = 12
-   **Best model**: Multiplicative seasonality + Damped trend (confirmed by AIC)
-   **Key insight**: AIC confirmed it was worth adding the damp parameter here (unlike Air Passengers)

### Air Miles Dataset

-   **Trend only**: no seasonality (annual data, frequency = 1)
-   **Models compared**: Holt linear vs. Holt damped
-   **Issue**: Only 24 observations → small sample → residual plots will NOT look perfect. This is expected.
-   **Best model**: Linear trend model had lower AIC

### Unemployment Dataset

-   **Negative trend** (decreasing) + **Seasonality**
-   **Best model**: Holt-Winters (all three components needed)
-   **ETS result**: Linear trend (no damping needed)

### US Accidents Dataset (1973–1978)

-   **Seasonality** present, periodicity = 12
-   **Trend**: unclear – could be no trend or a slight curve
-   **Best model**: Additive seasonality + Damped trend
    -   AIC barely higher than the "no trend" version, but RMSE was **much** better
-   **Key lesson**: Sometimes you must weigh AIC vs. RMSE and use your judgment

# Summary Points

-   🔍 **Always start by plotting** the data and identifying trend and seasonality visually.
-   ➕ or ✖️ **Additive vs. Multiplicative**: If seasonality grows over time → multiplicative. If constant → additive.
-   📊 **Decompose the series** and check the residuals for randomness, normality, and constant variance.
-   📏 **Standardise residuals** (mean=0, sd=1) so you can use the ±2 rule for the 95% boundary.
-   🤖 Use `ets()` in R as a **starting hint**, then try other models manually.
-   📉 **AIC** balances fit vs. complexity. Adding a parameter is only worth it if the error reduction is large.
-   ✅ After selecting a model, always **verify the final residuals** – good residuals = trustworthy forecasts.
-   📝 The exam will ask you to apply this full pipeline to a dataset (open book).
