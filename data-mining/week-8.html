2 Collaborative Filtering

2.1 Core Explanation

Collaborative filtering is a type of recommender system. Instead of looking at the content of items (like genre or description), it looks at the behaviour and preferences of users. The core idea is:

“If two people have rated many things similarly, they probably have similar taste – and we can use one person’s ratings to predict the other’s.”

There are two main flavours:

  • User-based collaborative filtering (UBCF): Find users who are similar to the target user. Use their ratings to predict what the target user would rate unseen items. Think of it as “people like you also liked…”
  • Item-based collaborative filtering (IBCF): Find items that tend to be rated similarly by many users. If two movies consistently receive similar ratings, seeing a rating for one lets you predict the other.

2.1.1 Example

User Toy Story GoldenEye Four Rooms Get Shorty Copycat
1 5 3 4 ?? 3
2 – 4 – 4 4
3 3 – 3 – –
4 – 5 – 4 ???
5 4 – – – –
6 – 3 – – –

Users 2 and 4 rate GoldenEye and Get Shorty similarly → we can infer User 4’s rating for Copycat is likely between 4 and 5.

2.2 Walkthrough in R (MovieLense Dataset)

2.2.1 Load and Explore

library(recommenderlab)
set.seed(42)
data(MovieLense)
# 943 users, 1664 movies
# rating = 0 means no rating was given

2.2.2 Filter the Data

Only keep users who rated ≥ 20 movies, and movies rated by ≥ 50 users. This removes sparse entries that would make predictions unreliable.

ratings = MovieLense[rowCounts(MovieLense) >= 20,
                     colCounts(MovieLense) >= 50]
dim(ratings)  # [1] 929 601

2.2.3 Normalise Ratings

Different users have different rating tendencies — some always rate things highly, others always rate things low. Normalisation centres each user’s ratings around zero so that these personal biases don’t distort comparisons.

norm_ratings = recommenderlab::normalize(ratings)

2.2.4 Split into Train / Test

The evaluationScheme() function splits data so the model can be trained and then evaluated. Key parameters:

  • given = 10: Each test user provides 10 known ratings for the model to use.
  • goodRating = 3: Ratings ≥ 3 count as a “good” recommendation.
  • k = 3: Run the evaluation 3 times (cross-validation).
evalScheme = evaluationScheme(data = ratings,
                              method = "split",
                              train = 0.9,
                              given = 10,
                              goodRating = 3,
                              k = 3)

2.2.5 Train and Evaluate Models

User-based (UBCF):

recommender = Recommender(data = getData(evalScheme, "train"),
                          method = "UBCF")
predictions = predict(object = recommender,
                      newdata = getData(evalScheme, "known"),
                      n = 12, type = "ratings")

accuracy1 = calcPredictionAccuracy(x = predictions,
                                   data = getData(evalScheme, "unknown"),
                                   byUser = FALSE)
# RMSE: 1.24  |  MAE: 0.98

Item-based (IBCF):

recommender = Recommender(data = getData(evalScheme, "train"),
                          method = "IBCF")
# RMSE: 1.24  |  MAE: 0.91  ← slightly better

2.2.6 Key Accuracy Metrics

Metric Meaning
MAE (Mean Absolute Error) Average prediction error in rating units. MAE ≈ 0.98 means predictions are off by ~1 star on average.
RMSE (Root Mean Squared Error) Similar to MAE but penalises large errors more heavily.
MSE Mean Squared Error – RMSE²

3 Dimensionality Reduction

3.1 Core Explanation

Imagine a spreadsheet with 784 columns (one per pixel in an image). Most of those columns carry redundant information — neighbouring pixels tend to have similar values. Dimensionality reduction compresses this data into fewer columns while trying to preserve as much of the meaningful variation as possible.

Key insight: “Dimensions” = columns = features. Fewer dimensions → faster models, less memory, less noise.

3.1.1 Principal Component Analysis (PCA)

PCA is the most widely used dimensionality reduction technique. Here’s how it works, step by step:

  1. Imagine your data as a cloud of points in multi-dimensional space.
  2. Find the line (direction) through that cloud that captures the most spread — this is the first principal component (PC1).
  3. Find the next direction that is perpendicular (orthogonal) to PC1 and captures the second-most spread → PC2.
  4. Repeat. Each component captures less and less variance.
  5. You can then keep only the top N components that together explain a sufficient percentage of variance (e.g., 80%).

PCA is related to linear regression in its use of least-squares fitting — both minimise the squared distance from points to a line.

3.2 MNIST Dataset

The MNIST database is a classic benchmark dataset: 70,000 greyscale images (28×28 pixels = 784 pixel columns) of handwritten digits 0–9. It is commonly used to test image classification and dimensionality reduction methods.

digits <- read.csv("data/train.csv")
dim(digits)  # 42000 rows × 785 columns (1 label + 784 pixels)

3.3 Applying PCA to MNIST

3.3.1 Step 1: Filter to One Digit Class

digits_5 <- digits[digits$label == 5, ]
digits_5$label <- NULL

# Remove pixels with zero variance (always black — no information)
digits_5b <- digits_5[, apply(digits_5, 2, var, na.rm = TRUE) != 0]

3.3.2 Step 2: Run PCA

pca <- prcomp(digits_5b, center = TRUE, scale. = TRUE)
s <- summary(pca)
cumprop <- s$importance[3, ]  # Cumulative proportion of variance explained
plot(cumprop, type = "l", main = "Cumulative sum", xlab = "PCA component")

The cumulative variance plot rises steeply at first (the first few components explain a lot) and then flattens (adding more components gives diminishing returns).

3.3.3 Step 3: Choose How Many Components

# How many components are needed to explain 80% of variance?
num_cols <- min(which(cumprop > 0.8))
num_cols   # 82 — only 82 components needed instead of 600+!

82 components explain 80% of the variance in ~600 original pixel columns. That is an enormous reduction in data size.

3.4 Key Formula

\[\text{Cumulative Variance Explained} = \sum_{i=1}^{k} \frac{\lambda_i}{\sum_{j} \lambda_j}\]

Where \(\lambda_i\) is the eigenvalue (variance) of the \(i\)-th principal component.

4 Outlier Analysis (Anomaly Detection)

4.1 Core Explanation

Outlier analysis (also called anomaly detection) finds data points that look very different from the rest. In a dataset of handwritten 5s, an outlier might be a particularly unusual or messy version of the digit — or even a mislabelled image.

Uses in practice: - Detecting fraudulent financial transactions - Identifying failing machinery - Flagging data entry errors

Outliers can sometimes be removed to improve model performance, or they can be the signal you care about (fraud detection).

4.2 Method 1: PCA Reconstruction Error

The idea: apply PCA to compress the data, then reconstruct it back to the original dimensions. If a data point reconstructs accurately → it looks like typical data. If reconstruction error is high → it is unusual.

preds <- predict(pca, digits_5b)

# Calculate MAE for each data point
error <- vector()
for (n in 1:1000) {
  actual_row <- digits_5b[n, ]
  pred_row   <- preds[n, ]
  error <- c(error, sum(abs(actual_row - pred_row)) / length(actual_row))
}

dfError <- data.frame(id = seq(1:length(error)), mae = error)
  • Low MAE → looks like a typical “5” (clean, standard handwriting).
  • High MAE → atypical “5” (very stylised, rotated, or possibly mislabelled as a 6).

4.3 Method 2: One-Class SVM

Support Vector Machines (SVM) are usually used for classification between two classes. A one-class SVM learns only from normal data and then flags any point that doesn’t fit the learned boundary as an outlier.

library(e1071)
svm.model <- svm(digits_5b, y = NULL,
                 type = 'one-classification',
                 nu = 0.10,       # ~10% of points expected to be outliers
                 scale = TRUE,
                 kernel = "radial")

svm.predtrain <- predict(svm.model, digits_5b)
table(svm.predtrain)
# FALSE: 383 outliers detected
# TRUE:  3412 normal points

The nu parameter controls how many outliers you expect (0.10 = allow up to 10%).

5 Graph Analytics

5.1 Core Explanation

Graph analytics (network analysis) studies structures made up of:

  • Nodes (also called vertices): The entities — e.g., people, web pages, towns.
  • Edges (also called links or relationships): The connections between entities — e.g., friendships, hyperlinks, roads.

Both nodes and edges can carry additional data (attributes), such as a user’s age (node attribute) or the distance between two towns (edge attribute/weight).

5.2 Graph Properties

Property Description Example
Connected vs Disconnected Is there a path between every pair of nodes? A social network is usually connected
Weighted vs Unweighted Do edges have a numerical value? Road distances between cities (weighted)
Directed vs Undirected Do relationships have a direction (A→B ≠ B→A)? Twitter follows (directed) vs Facebook friends (undirected)
Cyclic vs Acyclic Can a path start and end at the same node? A route map with loops is cyclic

5.3 Creating a Simple Graph in R

library(igraph)
g1 <- make_graph(edges = c(1,2, 2,3, 3,4, 4,1), directed = FALSE)
plot(g1, vertex.size = 40, edge.width = 2, edge.color = "blue")

5.4 Types of Graph Algorithms

  • Pathfinding: Find the shortest route through nodes. Classic example: the Travelling Salesman Problem.
  • Centrality: Identify the most important nodes in a network (e.g., most influential person in a social network).
  • Community Detection: Find clusters/groups within the network — similar to clustering but on relational data.

5.5 Real-World Applications

  • Marketing attribution: Tracing the sequence of adverts a user sees before a purchase.
  • Customer purchase modelling: Predicting what a customer will buy next based on purchase sequences.
  • Fraud network detection: Finding suspicious clusters of transactions.
  • Disease spread modelling: Mapping how an infection moves through a population.
  • Internet search engines: PageRank (see below).

5.6 PageRank

PageRank was the algorithm that powered Google’s original search engine. It estimates the importance of a web page by:

  1. Counting how many other pages link to it.
  2. Weighing those links by the importance of the linking page (a link from bbc.com is worth far more than a link from an obscure site).
  3. Iterating thousands of times until the scores stabilise.

This means a page with fewer but higher-quality inbound links can rank higher than a page with many low-quality links.

# PageRank can be computed in igraph
page_rank(g1)

6 PCA as a Pre-processing Step for Classification

6.1 Why Do This?

Running a classifier on 784 raw pixel columns is slow and can suffer from the curse of dimensionality (too many features, not enough signal per feature). By applying PCA first, you:

  • Reduce the number of input features dramatically.
  • Remove noise (low-variance components).
  • Speed up training significantly.
  • Often improve accuracy.

6.2 Full Workflow

6.2.1 1. Load and Filter Data

digits <- read.csv("data/train.csv")

# Keep only digits 5 and 6
data <- digits[digits$label %in% c(5, 6), ]
data$label <- as.factor(data$label)
# 3795 fives, 4137 sixes → 7932 total

6.2.2 2. Train/Test Split

Critical rule: Always split before applying PCA. If you apply PCA to the full dataset and then split, the test set influences the PCA — this is called data leakage and will give you falsely optimistic results.

set.seed(42)
input  <- setdiff(names(data), "label")
nobs   <- nrow(data)
train  <- sample(nobs, 0.9 * nobs)
test   <- setdiff(seq_len(nobs), train)

train_data <- data[train, ]
test_data  <- data[test,  ]

6.2.3 3. Apply PCA to Training Data Only

# Remove zero-variance pixels
train_data <- train_data[, apply(train_data, 2, var, na.rm = TRUE) != 0]
input <- setdiff(names(train_data), "label")

pca     <- prcomp(train_data[, input], center = TRUE, scale. = TRUE)
s       <- summary(pca)
cumprop <- s$importance[3, ]

# Find number of components for 80% variance
num_cols <- min(which(cumprop > 0.8))
# num_cols = 96  →  96 components explain 80% of variance

6.2.4 4. Project Both Sets onto PCA Space

# Training set: take first 96 principal components
train_pca <- as.data.frame(pca$x[, 1:num_cols])
train_pca$label <- train_data$label

# Test set: project using the SAME PCA transformation from train
test_pca <- as.data.frame(predict(pca, test_data[, input]))
test_pca$label <- test_data$label
test_pca <- test_pca[, names(train_pca)]  # keep same columns

6.2.5 5. Train a Logistic Regression Model

logistic_model <- glm(formula = label ~ .,
                      family = binomial(link = "logit"),
                      data = train_pca)

6.2.6 6. Evaluate

pr <- predict(logistic_model, newdata = test_pca, type = "response")
pr <- ifelse(pr < 0.5, 5, 6)
pr <- factor(pr)

tab <- table(pr, test_pca$label)
accuracy <- 100 * sum(diag(tab)) / sum(tab)
print(tab)
# Accuracy ≈ 97.4%

Confusion matrix result:

Actual 5 Actual 6
Predicted 5 379 10
Predicted 6 11 394

Only 21 misclassifications out of 794 test samples — excellent performance with greatly reduced data.

7 Summary Points

  • Collaborative filtering makes recommendations by finding similar users (UBCF) or similar items (IBCF), without needing to know anything about item content.
  • Normalisation of ratings is essential before collaborative filtering to correct for individual rating biases.
  • PCA reduces the number of columns in your data while retaining most of the useful variance. Only ~82 components were needed to represent 80% of the variance in ~600 MNIST pixel columns.
  • Outlier detection via PCA works by measuring how well each data point can be reconstructed after compression — high error = unusual point.
  • One-class SVM is an alternative outlier detection method that learns a boundary around normal data; anything outside is flagged as an anomaly.
  • Graphs consist of nodes (entities) and edges (relationships). Key properties are connectedness, direction, weight, and cyclicity.
  • PageRank ranks nodes (web pages) by the quality and quantity of incoming links — a core concept behind modern search engines.
  • Always split data before applying PCA to avoid data leakage. Apply the PCA transform fitted on training data to your test data.
  • With 96 PCA components (~80% variance retained), a logistic regression classifier achieved ~97.4% accuracy distinguishing digits 5 and 6.

# Graph Analytics

## Core Explanation

**Graph analytics** (network analysis) studies structures made up of:

- **Nodes** (also called vertices): The *entities* — e.g., people, web pages, towns.
- **Edges** (also called links or relationships): The *connections* between entities — e.g., friendships, hyperlinks, roads.

Both nodes and edges can carry additional data (**attributes**), such as a user's age (node attribute) or the distance between two towns (edge attribute/weight).

## Graph Properties

| Property | Description | Example |
|----------|-------------|---------|
| **Connected vs Disconnected** | Is there a path between every pair of nodes? | A social network is usually connected |
| **Weighted vs Unweighted** | Do edges have a numerical value? | Road distances between cities (weighted) |
| **Directed vs Undirected** | Do relationships have a direction (A→B ≠ B→A)? | Twitter follows (directed) vs Facebook friends (undirected) |
| **Cyclic vs Acyclic** | Can a path start and end at the same node? | A route map with loops is cyclic |

## Creating a Simple Graph in R

```r
library(igraph)
g1 <- make_graph(edges = c(1,2, 2,3, 3,4, 4,1), directed = FALSE)
plot(g1, vertex.size = 40, edge.width = 2, edge.color = "blue")
```

## Types of Graph Algorithms

- **Pathfinding:** Find the shortest route through nodes. Classic example: the *Travelling Salesman Problem*.
- **Centrality:** Identify the most *important* nodes in a network (e.g., most influential person in a social network).
- **Community Detection:** Find clusters/groups within the network — similar to clustering but on relational data.

## Real-World Applications

- **Marketing attribution:** Tracing the sequence of adverts a user sees before a purchase.
- **Customer purchase modelling:** Predicting what a customer will buy next based on purchase sequences.
- **Fraud network detection:** Finding suspicious clusters of transactions.
- **Disease spread modelling:** Mapping how an infection moves through a population.
- **Internet search engines:** PageRank (see below).

## PageRank

**PageRank** was the algorithm that powered Google's original search engine. It estimates the importance of a web page by:

1. Counting how many other pages link to it.
2. Weighing those links by the importance of the *linking* page (a link from bbc.com is worth far more than a link from an obscure site).
3. Iterating thousands of times until the scores stabilise.

This means a page with fewer but *higher-quality* inbound links can rank higher than a page with many low-quality links.

```r
# PageRank can be computed in igraph
page_rank(g1)
```

# PCA as a Pre-processing Step for Classification

## Why Do This?

Running a classifier on 784 raw pixel columns is slow and can suffer from the **curse of dimensionality** (too many features, not enough signal per feature). By applying PCA first, you:

- Reduce the number of input features dramatically.
- Remove noise (low-variance components).
- Speed up training significantly.
- Often *improve* accuracy.

## Full Workflow

### 1. Load and Filter Data

```r
digits <- read.csv("data/train.csv")

# Keep only digits 5 and 6
data <- digits[digits$label %in% c(5, 6), ]
data$label <- as.factor(data$label)
# 3795 fives, 4137 sixes → 7932 total
```

### 2. Train/Test Split

> **Critical rule:** Always split *before* applying PCA. If you apply PCA to the full dataset and then split, the test set influences the PCA — this is called **data leakage** and will give you falsely optimistic results.

```r
set.seed(42)
input  <- setdiff(names(data), "label")
nobs   <- nrow(data)
train  <- sample(nobs, 0.9 * nobs)
test   <- setdiff(seq_len(nobs), train)

train_data <- data[train, ]
test_data  <- data[test,  ]
```

### 3. Apply PCA to Training Data Only

```r
# Remove zero-variance pixels
train_data <- train_data[, apply(train_data, 2, var, na.rm = TRUE) != 0]
input <- setdiff(names(train_data), "label")

pca     <- prcomp(train_data[, input], center = TRUE, scale. = TRUE)
s       <- summary(pca)
cumprop <- s$importance[3, ]

# Find number of components for 80% variance
num_cols <- min(which(cumprop > 0.8))
# num_cols = 96  →  96 components explain 80% of variance
```

### 4. Project Both Sets onto PCA Space

```r
# Training set: take first 96 principal components
train_pca <- as.data.frame(pca$x[, 1:num_cols])
train_pca$label <- train_data$label

# Test set: project using the SAME PCA transformation from train
test_pca <- as.data.frame(predict(pca, test_data[, input]))
test_pca$label <- test_data$label
test_pca <- test_pca[, names(train_pca)]  # keep same columns
```

### 5. Train a Logistic Regression Model

```r
logistic_model <- glm(formula = label ~ .,
                      family = binomial(link = "logit"),
                      data = train_pca)
```

### 6. Evaluate

```r
pr <- predict(logistic_model, newdata = test_pca, type = "response")
pr <- ifelse(pr < 0.5, 5, 6)
pr <- factor(pr)

tab <- table(pr, test_pca$label)
accuracy <- 100 * sum(diag(tab)) / sum(tab)
print(tab)
# Accuracy ≈ 97.4%
```

**Confusion matrix result:**

|  | Actual 5 | Actual 6 |
|--|----------|----------|
| **Predicted 5** | 379 | 10 |
| **Predicted 6** | 11 | 394 |

Only 21 misclassifications out of 794 test samples — excellent performance with greatly reduced data.

# Summary Points

- **Collaborative filtering** makes recommendations by finding similar users (UBCF) or similar items (IBCF), without needing to know anything about item content.
- **Normalisation** of ratings is essential before collaborative filtering to correct for individual rating biases.
- **PCA** reduces the number of columns in your data while retaining most of the useful variance. Only ~82 components were needed to represent 80% of the variance in ~600 MNIST pixel columns.
- **Outlier detection via PCA** works by measuring how well each data point can be reconstructed after compression — high error = unusual point.
- **One-class SVM** is an alternative outlier detection method that learns a boundary around normal data; anything outside is flagged as an anomaly.
- **Graphs** consist of nodes (entities) and edges (relationships). Key properties are connectedness, direction, weight, and cyclicity.
- **PageRank** ranks nodes (web pages) by the quality and quantity of incoming links — a core concept behind modern search engines.
- **Always split data before applying PCA** to avoid data leakage. Apply the PCA transform fitted on training data to your test data.
- With 96 PCA components (~80% variance retained), a logistic regression classifier achieved ~97.4% accuracy distinguishing digits 5 and 6.
