1 Lecture Overview

This week focuses on recommendation systems, including:

  • Association rules (apriori)
  • Customer segmentation
  • Collaborative filtering
  • Ethical considerations
  • Hyperparameters and optimisation
  • Graph Analytics

2 Key Concepts

2.1 Recommendation Systems

A recommendation system suggests items to users based on data.

2.1.1 Types:

  • Association rule-based (Apriori)
  • Content-based filtering
  • Collaborative filtering

2.2 Association Rules (Apriori)

Used to find relationships between items.

2.2.1 Example:

Customers who buy bread also buy butter

2.2.2 Key measures:

  • Support → how often items appear together
  • Confidence → probability of buying Y given X
  • Lift → strength of the relationship

👉 In practice: - focus on high lift rules - filter out weak or obvious relationships

2.3 Customer Segmentation

Before applying rules, customers are often grouped.

2.3.1 Why?

Different groups behave differently.

2.3.2 Example:

  • Students → low budget
  • Professionals → higher spending

👉 Apply recommendation rules within each segment

2.4 Collaborative Filtering

This is a key concept for exams.

2.4.1 Definition

Collaborative filtering makes recommendations based on similar users.

2.5 How it works

  1. Collect user preferences (ratings, purchases, clicks)
  2. Find users with similar behaviour
  3. Recommend items liked by similar users

2.6 Example

  • User A and User B both like:
    • Netflix shows A, B, C
  • User B also likes:
    • Show D

👉 Recommend Show D to User A

2.7 Types of Collaborative Filtering

2.7.1 1. User-based

  • Find similar users
  • Recommend based on their preferences

2.7.2 2. Item-based

  • Find similar items
  • Recommend similar items to what user already likes

2.8 Advantages

  • Simple and intuitive
  • Works well with large datasets
  • No need for item features

2.9 Limitations

  • Cold start problem (new users/items)
  • Requires lots of data
  • Can reinforce bias

2.10 Ethics in Recommendation Systems

Important exam topic ⚠️

2.10.1 Issues:

  • Privacy concerns
  • Bias and discrimination
  • Filter bubbles (only showing similar content)
  • Manipulation of user behaviour

2.11 Hyperparameters

Settings that control how a model learns.

2.11.1 Examples:

  • Number of clusters (k in k-means)
  • Number of neighbours in collaborative filtering

2.12 Hyperparameter Optimisation

Choosing the best values for hyperparameters.

2.12.1 Methods:

  • Trial and error
  • Grid search
  • Cross-validation

2.13 Graph Analytics

Graph analytics (or network analysis) is the study of networks or graphs. A network / graph has nodes and edges between them. The nodes and the edges may both have attributes.

2.13.1 Can be used in:

  • Marketing attribution analysis: this analyses the adverts shown to a user and the activities. It may show that indirect activities that are in an earlier step can have casual properties.

  • Customer purchase modeling: By modeling the sequence of purchases as a graph, you can generate recommendations of what customers are likely to buy next.

  • Fraud network detection - internet search engines: graph analytics are a core part of internet search algorithms. Relationship that exists in the transaction.

  • Disease cluster analysis: can be used to model and the spread of diseases

3 R Example (Collaborative Filtering)

Code 
library(recommenderlab)

data("MovieLense")

# View dataset
MovieLense
943 x 1664 rating matrix of class 'realRatingMatrix' with 99392 ratings.
Code 
# Build a recommendation model
model <- Recommender(MovieLense, method = "UBCF")

# Generate recommendations
rec <- predict(model, MovieLense[1:5], n = 3)

rec
Recommendations as 'topNList' with n = 3 for 5 users.
