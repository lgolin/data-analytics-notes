Case Analysis Study Guide

📝 How to use this guide: Each exercise follows the same pattern — plot → decompose → fit models → compare → forecast. Master the pattern and you can handle any dataset. Leave space below each section for your own annotations!

1 Background: The Toolkit You Need

Before diving in, load the main package we use throughout:

Code 
library(forecast)
library(ggplot2)

1.1 What is a Time Series?

A time series is just a sequence of numbers recorded over time — like monthly wine sales, yearly unemployment rates, or daily stock prices. The goal is to understand the pattern and predict the future.

Every time series can be broken into up to four components:

Component What it means Example
Trend (T) Long-term direction — going up or down? Wine sales growing over decades
Seasonality (S) Regular repeating pattern within a year More wine sold in December every year
Cycle (C) Longer-term waves (usually folded into Trend) Economic booms and busts
Residual / Random (R) What’s left after removing the above — noise Unpredictable spikes

1.2 Additive vs Multiplicative: The Big Choice

This is one of the most important decisions you’ll make.

\[y_t = S_t + T_t + R_t \quad \text{(Additive)}\]

\[y_t = S_t \times T_t \times R_t \quad \text{(Multiplicative)}\]

How to decide?

  • Additive → seasonal peaks stay roughly the same height over time (flat seasonality)
  • Multiplicative → seasonal peaks grow bigger as the series grows (expanding seasonality)

🧠 Tip for the exam: Look at your plot. If the “wobbles” get bigger as time goes on → Multiplicative. If the “wobbles” stay the same size → Additive.

1.3 The Classical Models: SES, Holt, Holt-Winters

Think of these as three levels of complexity:

Model R Function When to use
Simple Exponential Smoothing (SES) ses() No trend, no seasonality
Holt (Double ES) holt() Has trend, no seasonality
Holt-Winters (Triple ES) hw() Has trend AND seasonality
ETS (automatic) ets() Let R decide the best combination

1.4 How to Compare Models (GOF Measures)

After fitting models, use these to decide which is best:

  • RMSE / MAPE → lower = better fit to the data (from accuracy())
  • AIC / BIC → lower = better balance of fit AND simplicity (from summary())

🏆 Francisco’s golden rule: Prefer AIC/BIC over RMSE alone — they penalise you for making the model too complex. A simpler model that fits almost as well is usually better.

2 Exercise 1: Wine Industry (wineind)

2.1 The Questions

  1. Find the moving average. Use the appropriate period. Plot it.
  2. Decompose the time series using different commands. Comment.
  3. Fit 3 different classical models. Compare them.
  4. Forecast using the best model.

2.2 Step 1 — Plot the Raw Data

Code 
# Always start by looking at your data!
plot(wineind)

What to look for when you plot:

  • Is there an overall upward or downward trend?
  • Do you see repeating peaks and valleys (seasonality)?
  • Do those peaks get bigger over time (multiplicative) or stay the same size (additive)?

🍷 Interpretation of wineind: The wine dataset shows monthly wine sales in Australia. You should see a clear seasonal pattern (sales spike regularly — think holiday periods) and a possible upward trend. The seasonal spikes appear to grow with the level of the series, suggesting a multiplicative model will be more appropriate.

2.3 Step 2 — Moving Average (Smoothing the noise)

A moving average smooths out the short-term noise so you can see the trend more clearly. Think of it like drawing a smooth line through a messy scribble.

For monthly data, the period is 12 (12 months in a year).

Code 
# Calculate and plot the 12-period moving average
sm <- ma(wineind, order = 12)

plot(wineind)
lines(sm, col = "red", lwd = 2)  # overlay the smooth line in red

💡 Why order = 12? Because wineind is monthly data — one full seasonal cycle = 12 months. For quarterly data, you’d use order = 4.

What to say about this plot: The red line shows the underlying trend after removing seasonal noise. It reveals whether the series is generally increasing, decreasing, or flat.

2.4 Step 3 — Decompose the Series

Decomposition splits the series into its components so you can study each one separately.

Code 
# Method 1: Classic decompose (multiplicative, since seasonality grows with level)
autoplot(decompose(wineind, type = "multiplicative"))

# Look at the random component — it should look like random noise
rr <- decompose(wineind, type = "multiplicative")$random
hist(rr)           # should look like a bell curve (normal distribution)
plot(rr, type = "p")  # should look like scattered random dots
Code 
# Method 2: STL decomposition (more flexible, handles changing seasonality)
autoplot(stl(wineind, s.window = "periodic"))
Code 
# Method 3: MSTL (Multiple STL — good for detecting multiple seasonal patterns)
wineind %>% mstl() %>% autoplot()

🧠 How to interpret the decomposition panels:

  • Top panel (Data): The original series
  • Seasonal panel: The repeating pattern — does it look consistent?
  • Trend panel: The overall direction — going up, down, or flat?
  • Remainder/Random panel: What’s left — ideally this looks like random noise with no pattern. If you see a clear pattern here, your model hasn’t captured everything!

For the exam: Always comment on whether the residuals look random. If they don’t, the model is missing something.

2.5 Step 4 — Fit 3 Classical Models

Since wineind has both trend and seasonality, we use Holt-Winters (hw()).

Code 
# Model 1: Holt-Winters WITHOUT damping (assumes trend continues forever)
h1 <- hw(wineind, damped = FALSE)
summary(h1)
autoplot(h1)
Code 
# Model 2: Holt-Winters WITH damping (trend slows down over time — often more realistic)
h2 <- hw(wineind, damped = TRUE)
summary(h2)
autoplot(h2)
Code 
# Model 3: STLF (STL decomposition + ETS on seasonally adjusted data)
autoplot(stlf(wineind, s.window = "periodic"))

What is “damped”? A damped trend means R assumes the trend will slow down and level off in the future, rather than continuing at the same rate forever. In real life, things rarely grow (or shrink) at a constant rate forever — so damped models are often more realistic.

🧪 Comparing models: Use summary() and look at AIC/BIC. The model with the lowest AIC wins. Also check RMSE from accuracy().

2.6 Step 5 — Automatic ETS (Let R Choose)

Code 
# Let R find the best ETS model automatically
fit <- ets(wineind)   # "ZZZ" means auto-select all components
summary(fit)
plot(fit)

Reading the ETS output:

The model name looks like ETS(M, Ad, M):

  • 1st letter = Error type: A = Additive, M = Multiplicative
  • 2nd letter = Trend: N = None, A = Additive, Ad = Additive Damped, M = Multiplicative
  • 3rd letter = Season: N = None, A = Additive, M = Multiplicative

📌 ETS(M, Ad, M) means: Multiplicative errors, Damped additive trend, Multiplicative seasonality. This is a common “best” model for data with growing seasonality and a trend that levels off.

2.7 Step 6 — Forecast with the Best Model

Code 
# Forecast 12 months ahead (= 1 year for monthly data)
forecast(fit, h = 12)
plot(forecast(fit, h = 12))

💡 Rule of thumb for h: Use h = frequency × n_years. For monthly data, h = 12 = 1 year ahead. For quarterly data, h = 4 = 1 year ahead.

Interpreting the forecast plot:

  • The dark blue line = point forecast (our best guess)
  • The shaded areas = confidence intervals (80% and 95% — the wider they are, the more uncertain we are)

3 Exercise 2: US Civilian Unemployment Rate

3.1 The Questions

  1. Visual and descriptive analysis. Transform to TS and plot.
  2. Forecast next year using Simple Exponential Smoothing (ses()).
  3. Find the best ETS model automatically, forecast 12 months, plot.
  4. Summarise: seasonality? trend? what type of model?
  5. Use ets() to estimate models with and without trend, compare.

3.2 Step 1 — Load and Plot the Data

Code 
# Read in the unemployment CSV (monthly data, 2012-2017)
df <- read.csv("unemployment.csv")

# Convert to a time series object
# start = c(year, month), frequency = 12 for monthly
unempl <- ts(df[, 1], start = c(2012, 1), frequency = 12)

# Plot it
plot(unempl)

🔍 What to look for: You should see that unemployment was relatively high in 2012 and declined steadily toward 2017 — this reflects the US economic recovery after the 2008 financial crisis. There may also be very mild seasonal patterns.

3.3 Step 2 — Simple Exponential Smoothing (SES)

SES is the simplest forecasting method. It just takes a weighted average of past values, giving more weight to recent observations.

Code 
# Fit SES and forecast 12 months ahead
fcast_ses <- ses(unempl, h = 12)
summary(fcast_ses)
plot(fcast_ses)

What alpha (α) tells you:

  • α close to 1 → model relies mostly on the most recent observation (short memory)
  • α close to 0 → model uses a long history of past data (long memory)

⚠️ Limitation of SES: Since it has no trend component, its forecast is a flat horizontal line into the future. If your data has a trend (like unemployment declining), SES will underperform. This is why we check other models!

3.4 Step 3 — Let ETS Choose the Best Model

Code 
# Let R auto-select the best ETS model
fit_ets_default <- ets(unempl)   # equivalent to model = "ZZZ"
fit_ets_default                  # prints the chosen model type
summary(fit_ets_default)

# Forecast 12 months
fcast_ets_default <- forecast(fit_ets_default, h = 12)
plot(fcast_ets_default)

3.5 Step 4 — Interpret the Model

After running summary(fit_ets_default), answer these questions:

Question Where to look What it means
What type of model? Model name e.g. ETS(A,A,N) Error / Trend / Season
Is there a trend? 2nd letter — A, M or N? N = no trend
Is there seasonality? 3rd letter — A, M or N? N = no seasonality
Is the trend damped? damped = TRUE in output or d in name Trend slows down
How good is the fit? AIC, RMSE from accuracy() Lower = better

💬 Sample interpretation: “The model selected by R is ETS(A,N,N) — additive errors, no trend, no seasonality. This is equivalent to Simple Exponential Smoothing. Alpha = 0.85, indicating the model has a short memory and relies heavily on recent unemployment figures. The AIC is [value].”

3.6 Step 5 — Compare Models With and Without Trend

Code 
# Model WITH damped trend
fit_ets_damped_trend <- ets(unempl, damped = TRUE)
fcast_ets_damped_trend <- forecast(fit_ets_damped_trend, h = 12)
plot(fcast_ets_damped_trend, main = "With Damped Trend")

# Model WITHOUT trend (force no trend component)
fit_ets_no_trend <- ets(unempl, model = "ZNZ")   # Z=auto, N=no trend, Z=auto
fcast_ets_no_trend <- forecast(fit_ets_no_trend, h = 12)
plot(fcast_ets_no_trend, main = "No Trend")
Code 
# Compare all three with accuracy metrics
accuracy(fcast_ets_damped_trend)
accuracy(fcast_ets_default)
accuracy(fcast_ets_no_trend)

# Also compare AIC from summary
summary(fcast_ets_damped_trend)
summary(fcast_ets_no_trend)

📊 How to write up the comparison:

“Model A (with damped trend) has RMSE = X and AIC = Y. Model B (no trend) has RMSE = X2 and AIC = Y2. Since Model [A/B] has a lower AIC, it represents a better balance between fit quality and model complexity, and is therefore preferred.”

4 Exercise 3: Airmiles Dataset

4.1 The Questions

  1. Find the moving average. Use appropriate period. Plot.
  2. Decompose using decompose, stl, mstl. Comment.
  3. Fit 3 classical models. Compare.
  4. Forecast using best model.

4.2 Step 1 — Plot and Moving Average

Code 
# airmiles is ANNUAL data (frequency = 1), so NO seasonal moving average
# We still plot to see the trend
plot(airmiles)

# Since it's annual, there's no seasonal period to average
# mstl handles this gracefully
airmiles %>% mstl() %>% autoplot()

✈️ Interpretation of airmiles: This dataset records revenue passenger-miles for US airlines. You should see a clear upward trend — air travel grew dramatically after WWII. There is no seasonality because the data is annual. This means we only need SES or Holt (no Holt-Winters needed).

4.3 Step 2 — Fit 3 Models

Since airmiles has a trend but no seasonality, use SES and Holt:

Code 
# Model 1: SES (no trend — expect this to be worst)
fcast_s <- ses(airmiles)
plot(fcast_s)
accuracy(fcast_s)
summary(fcast_s)
Code 
# Model 2: Holt (captures trend — should fit better)
fcast_h <- holt(airmiles)
plot(fcast_h)
accuracy(fcast_h)
summary(fcast_h)
Code 
# Model 3: Linear Regression on trend (off the record — extra insight)
fit_reg <- tslm(airmiles ~ trend)
summary(fit_reg)
plot(forecast(fit_reg, h = 20))

# Check the residuals
rr <- fit_reg$residuals
plot(rr, type = 'p')   # should look random

🧠 Why does Holt beat SES here? Because airmiles has a clear upward trend. SES ignores trend (its forecast is flat), while Holt explicitly models trend with a second parameter (beta). Always match the model to the data components you observe!

5 Exercise 4: US Accident Deaths (USAccDeaths)

5.1 The Questions

  1. Plot the data. What do you observe?
  2. Apply the best method based on your observation.
  3. Which model — additive or multiplicative?
  4. Use GOF measures (accuracy()) to support.
  5. Which model fits better? Use appropriate period.
  6. Forecast with all models, plot and compare.

5.2 Step 1 — Plot and Identify Components

Code 
plot(USAccDeaths)

🚗 Interpretation of USAccDeaths: Monthly accidental deaths in the US, 1973-1978. You should observe:

  • Seasonal pattern: Clear peaks in summer months (July) — more outdoor activity, more accidents. This repeats every year.
  • No strong long-term trend: The level appears roughly stable.
  • Seasonality appears roughly constant in size → suggests additive is reasonable, but check multiplicative too since the seasonal spikes track the overall level.

5.3 Step 2 — Decompose

Code 
# Full decomposition
plot(decompose(USAccDeaths))

# STL decomposition
USAccDeaths %>% stl(s.window = "periodic") %>% autoplot()
USAccDeaths %>% mstl() %>% autoplot()

5.4 Step 3 — Fit Models: Additive vs Multiplicative

Code 
# Model 1: Holt-Winters MULTIPLICATIVE seasonal
deaths.fcast1 <- hw(USAccDeaths, seasonal = "multiplicative", h = 48)
summary(deaths.fcast1)
plot(deaths.fcast1)
Code 
# Model 2: Holt-Winters ADDITIVE seasonal
deaths.fcast2 <- hw(USAccDeaths, seasonal = "additive", h = 48)
summary(deaths.fcast2)
plot(deaths.fcast2)
Code 
# Model 3: Holt-Winters ADDITIVE with DAMPED trend
deaths.fcast3 <- hw(USAccDeaths, seasonal = "additive", damped = TRUE, h = 48)
summary(deaths.fcast3)
plot(deaths.fcast3)

5.5 Step 4 — Compare All Models

Code 
accuracy(deaths.fcast1)   # Multiplicative
accuracy(deaths.fcast2)   # Additive         (lower AIC?)
accuracy(deaths.fcast3)   # Additive + Damped (lowest AIC?)

📊 What the lecturer expects you to say:

“Comparing the three models, Model 3 (additive with damped trend) has the lowest AIC = 1141, followed by Model 2 (additive, AIC = 1145). Despite having an extra parameter (phi for damping), the reduction in RMSE justifies the added complexity. However, since the AIC difference is small, the simpler additive model is also defensible. Multiplicative seasonality did not improve fit significantly for this dataset, suggesting the seasonal amplitude is relatively constant.”

⚠️ Note on GOF vs AIC: RMSE measures absolute error. AIC penalises complexity. Use AIC as your primary criterion when choosing between models. The model with MORE parameters will almost always have lower RMSE — but that doesn’t mean it’s better!

6 Exercise 5: EuroStock Markets (EuStockMarkets)

6.1 The Questions

  1. Plot EuStockMarkets. What do you observe?
  2. Implement ETS methods.
  3. Plot fitted lines on observations 200-300, compare to true TS.
  4. Are classical models useful for this type of data?

6.2 Step 1 — Plot and Convert

Code 
# EuStockMarkets has 4 stock indices — we use the first one (DAX)
EuStock <- ts(EuStockMarkets[, 1], start = c(1991, 130), frequency = 260)

# Plot it
plot(EuStock)
EuStock %>% mstl() %>% autoplot()

📈 Interpretation of EuStockMarkets: European stock market indices from 1991-1998. You should see:

  • A strong upward trend throughout the 1990s (bull market period)
  • No clear seasonality — stock prices don’t follow yearly seasonal patterns
  • High volatility and irregular movements
  • Frequency = 260 (trading days per year) → R’s ets() cannot handle frequency > 24, so seasonality is ignored

6.3 Step 2 — Fit ETS Models

Code 
# Note: ets() will warn that seasonality is ignored (frequency > 24)
ets(EuStock)

# Use stlf() if you want seasonal forecasts despite high frequency
stlf(EuStock)

6.4 Step 3 — Fit and Evaluate on Observations 200-300

This is about backtesting — training on early data and testing on later data.

Code 
# MODEL 1: SES
# Train on first 200 observations, forecast next 100
fit1 <- ses(EuStock[1:200], h = 100)
accuracy(fit1)
plot(fit1)
lines(EuStock[1:300])    # overlay the actual data

# How well did we predict obs 201-300?
accuracy(fit1, EuStock[201:300])
Code 
# MODEL 2: Holt (with trend)
fit2 <- holt(EuStock[1:200], h = 100)
accuracy(fit2)
plot(fit2)
lines(EuStock[1:300])

accuracy(fit2, EuStock[201:300])

6.5 Step 4 — The Big Question: Are Classical Models Useful Here?

💬 This is the most important part of Exercise 5 — the professor wants your interpretation!

Expected answer:

“Classical models (SES, Holt, Holt-Winters) are limited for financial stock market data. Stock prices exhibit highly irregular, volatile movements driven by external news, economic events, and investor sentiment — factors that classical exponential smoothing models cannot capture.

When we train on observations 1-200 and predict 201-300, the forecast errors are large. The actual stock price moves significantly away from the model’s prediction (confidence intervals are very wide or the series falls outside them entirely).

This suggests that ARIMA or SARIMA models (which we study later) or even more advanced financial models would be more appropriate for stock market data. Classical models work well for data with stable seasonal patterns and smooth trends, but struggle with high-volatility financial series.”

7 Summary: The Universal Workflow

Use this checklist on every dataset in the exam:

1. PLOT the raw data
   → Identify: trend? seasonality? additive or multiplicative?

2. DECOMPOSE
   → decompose() / stl() / mstl()
   → Check residuals: do they look random?

3. SELECT the right model
   → No trend, no season → SES
   → Trend only           → Holt
   → Trend + seasonality  → Holt-Winters (hw)
   → Not sure?            → ets("ZZZ") and let R decide

4. FIT multiple models (at least 3)
   → Try with/without damping
   → Try additive vs multiplicative seasonality

5. COMPARE with GOF measures
   → accuracy() → RMSE, MAPE
   → summary()  → AIC, BIC
   → Lower = better

6. SELECT best model and FORECAST
   → h = frequency × number of periods ahead
   → Interpret confidence intervals

7. CHECK residuals of final model
   → hist(residuals) → bell shaped?
   → plot(residuals, type='p') → random scatter?

8 Key Equations Reference

Additive decomposition: \[y_t = S_t + T_t + R_t\]

Multiplicative decomposition: \[y_t = S_t \times T_t \times R_t\]

Simple Exponential Smoothing (SES): \[\hat{y}_{t+1} = \alpha y_t + (1-\alpha)\hat{y}_t, \quad 0 < \alpha < 1\]

Reading ETS model codes:

ETS( Error , Trend , Season )
     A/M/Z   N/A/Ad/M/Z   N/A/M/Z

A = Additive
M = Multiplicative  
N = None
Z = Auto-select
d = Damped (written as "Ad")

Model comparison rule: \[\text{Lower AIC} = \text{Better model}\]

## Step 1 — Plot the Raw Data

```{r}
# Always start by looking at your data!
```

**What to look for when you plot:**

- Is there an overall **upward or downward trend**?
- Do you see **repeating peaks and valleys** (seasonality)?

> 🍷 **Interpretation of `wineind`:** The wine dataset shows monthly wine sales in Australia. You should see a clear **seasonal pattern** (sales spike regularly — think holiday periods) and a possible **upward trend**. The seasonal spikes appear to grow with the level of the series, suggesting a **multiplicative** model will be more appropriate.

---

## Step 2 — Moving Average (Smoothing the noise)

A **moving average** smooths out the short-term noise so you can see the trend more clearly. Think of it like drawing a smooth line through a messy scribble.

For **monthly data**, the period is **12** (12 months in a year).

```{r}
# Calculate and plot the 12-period moving average
sm <- ma(wineind, order = 12)

plot(wineind)
lines(sm, col = "red", lwd = 2)  # overlay the smooth line in red
```

> 💡 **Why order = 12?** Because `wineind` is monthly data — one full seasonal cycle = 12 months. For quarterly data, you'd use `order = 4`.

**What to say about this plot:** The red line shows the underlying trend after removing seasonal noise. It reveals whether the series is generally increasing, decreasing, or flat.

---

## Step 3 — Decompose the Series

Decomposition splits the series into its components so you can study each one separately.

```{r}
# Method 1: Classic decompose (multiplicative, since seasonality grows with level)
autoplot(decompose(wineind, type = "multiplicative"))

# Look at the random component — it should look like random noise
rr <- decompose(wineind, type = "multiplicative")$random
hist(rr)           # should look like a bell curve (normal distribution)
plot(rr, type = "p")  # should look like scattered random dots
```

```{r}
# Method 2: STL decomposition (more flexible, handles changing seasonality)
autoplot(stl(wineind, s.window = "periodic"))
```

```{r}
# Method 3: MSTL (Multiple STL — good for detecting multiple seasonal patterns)
wineind %>% mstl() %>% autoplot()
```

> 🧠 **How to interpret the decomposition panels:**
>
> - **Top panel (Data):** The original series
> - **Seasonal panel:** The repeating pattern — does it look consistent?
> - **Trend panel:** The overall direction — going up, down, or flat?
> - **Remainder/Random panel:** What's left — ideally this looks like random noise with no pattern. If you see a clear pattern here, your model hasn't captured everything!
>
> **For the exam:** Always comment on whether the residuals look random. If they don't, the model is missing something.

---

## Step 4 — Fit 3 Classical Models

Since `wineind` has both **trend** and **seasonality**, we use Holt-Winters (`hw()`).

```{r}
# Model 1: Holt-Winters WITHOUT damping (assumes trend continues forever)
h1 <- hw(wineind, damped = FALSE)
summary(h1)
autoplot(h1)
```

```{r}
# Model 2: Holt-Winters WITH damping (trend slows down over time — often more realistic)
h2 <- hw(wineind, damped = TRUE)
summary(h2)
autoplot(h2)
```

```{r}
# Model 3: STLF (STL decomposition + ETS on seasonally adjusted data)
autoplot(stlf(wineind, s.window = "periodic"))
```

**What is "damped"?**
A **damped trend** means R assumes the trend will slow down and level off in the future, rather than continuing at the same rate forever. In real life, things rarely grow (or shrink) at a constant rate forever — so damped models are often more realistic.

> 🧪 **Comparing models:** Use `summary()` and look at **AIC/BIC**. The model with the **lowest AIC** wins. Also check RMSE from `accuracy()`.

---

## Step 5 — Automatic ETS (Let R Choose)

```{r}
# Let R find the best ETS model automatically
fit <- ets(wineind)   # "ZZZ" means auto-select all components
summary(fit)
plot(fit)
```

**Reading the ETS output:**

The model name looks like `ETS(M, Ad, M)`:

- **1st letter** = Error type: `A` = Additive, `M` = Multiplicative
- **2nd letter** = Trend: `N` = None, `A` = Additive, `Ad` = Additive Damped, `M` = Multiplicative
- **3rd letter** = Season: `N` = None, `A` = Additive, `M` = Multiplicative

> 📌 `ETS(M, Ad, M)` means: Multiplicative errors, Damped additive trend, Multiplicative seasonality. This is a common "best" model for data with growing seasonality and a trend that levels off.

---

## Step 6 — Forecast with the Best Model

```{r}
# Forecast 12 months ahead (= 1 year for monthly data)
forecast(fit, h = 12)
plot(forecast(fit, h = 12))
```

> 💡 **Rule of thumb for h:** Use `h = frequency × n_years`. For monthly data, `h = 12` = 1 year ahead. For quarterly data, `h = 4` = 1 year ahead.

**Interpreting the forecast plot:**

- The **dark blue line** = point forecast (our best guess)
- The **shaded areas** = confidence intervals (80% and 95% — the wider they are, the more uncertain we are)

```{r}
# Read in the unemployment CSV (monthly data, 2012-2017)
```

> 🔍 **What to look for:** You should see that unemployment was relatively high in 2012 and **declined steadily** toward 2017 — this reflects the US economic recovery after the 2008 financial crisis. There may also be very mild seasonal patterns.

---

## Step 2 — Simple Exponential Smoothing (SES)

SES is the **simplest** forecasting method. It just takes a weighted average of past values, giving more weight to recent observations.

```{r}
# Fit SES and forecast 12 months ahead
fcast_ses <- ses(unempl, h = 12)
summary(fcast_ses)
plot(fcast_ses)
```

**What alpha (α) tells you:**

- **α close to 1** → model relies mostly on the most recent observation (short memory)
- **α close to 0** → model uses a long history of past data (long memory)

> ⚠️ **Limitation of SES:** Since it has no trend component, its forecast is a **flat horizontal line** into the future. If your data has a trend (like unemployment declining), SES will underperform. This is why we check other models!

---

## Step 3 — Let ETS Choose the Best Model

```{r}
# Let R auto-select the best ETS model
fit_ets_default <- ets(unempl)   # equivalent to model = "ZZZ"
fit_ets_default                  # prints the chosen model type
summary(fit_ets_default)

# Forecast 12 months
fcast_ets_default <- forecast(fit_ets_default, h = 12)
plot(fcast_ets_default)
```

---

## Step 4 — Interpret the Model

After running `summary(fit_ets_default)`, answer these questions:

| Question | Where to look | What it means |
|----------|--------------|---------------|
| What type of model? | Model name e.g. `ETS(A,A,N)` | Error / Trend / Season |
| Is there a trend? | 2nd letter — `A`, `M` or `N`? | `N` = no trend |
| Is there seasonality? | 3rd letter — `A`, `M` or `N`? | `N` = no seasonality |
| Is the trend damped? | `damped = TRUE` in output or `d` in name | Trend slows down |
| How good is the fit? | AIC, RMSE from `accuracy()` | Lower = better |

> 💬 **Sample interpretation:** "The model selected by R is ETS(A,N,N) — additive errors, no trend, no seasonality. This is equivalent to Simple Exponential Smoothing. Alpha = 0.85, indicating the model has a short memory and relies heavily on recent unemployment figures. The AIC is [value]."

## Step 5 — Compare Models With and Without Trend

```{r}
# Model WITH damped trend
fit_ets_damped_trend <- ets(unempl, damped = TRUE)
```

```{r}
summary(fcast_ets_damped_trend)
summary(fcast_ets_no_trend)
```

>
> "Model A (with damped trend) has RMSE = X and AIC = Y. Model B (no trend) has RMSE = X2 and AIC = Y2. Since Model [A/B] has a lower AIC, it represents a better balance between fit quality and model complexity, and is therefore preferred."

# Summary: The Universal Workflow

Use this checklist on every dataset in the exam:

```
1. PLOT the raw data
   → Identify: trend? seasonality? additive or multiplicative?

2. DECOMPOSE
   → decompose() / stl() / mstl()
   → Check residuals: do they look random?

3. SELECT the right model
   → No trend, no season → SES
   → Trend only           → Holt
   → Trend + seasonality  → Holt-Winters (hw)
   → Not sure?            → ets("ZZZ") and let R decide

4. FIT multiple models (at least 3)
   → Try with/without damping
   → Try additive vs multiplicative seasonality

5. COMPARE with GOF measures
   → accuracy() → RMSE, MAPE
   → summary()  → AIC, BIC
   → Lower = better

6. SELECT best model and FORECAST
   → h = frequency × number of periods ahead
   → Interpret confidence intervals

7. CHECK residuals of final model
   → hist(residuals) → bell shaped?
   → plot(residuals, type='p') → random scatter?
```

# Key Equations Reference

**Additive decomposition:**
$$y_t = S_t + T_t + R_t$$

**Multiplicative decomposition:**
$$y_t = S_t \times T_t \times R_t$$

**Simple Exponential Smoothing (SES):**
$$\hat{y}_{t+1} = \alpha y_t + (1-\alpha)\hat{y}_t, \quad 0 < \alpha < 1$$

**Reading ETS model codes:**

```
ETS( Error , Trend , Season )
     A/M/Z   N/A/Ad/M/Z   N/A/M/Z

A = Additive
M = Multiplicative  
N = None
Z = Auto-select
d = Damped (written as "Ad")
```

**Model comparison rule:**
$$\text{Lower AIC} = \text{Better model}$$
