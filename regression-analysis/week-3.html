2 Part 1: Missing Data

2.1 What Is Missing Data?

In real-world datasets, it is very common to find that some values are not recorded. These are called missing values. In R, missing values are stored as NA — short for “Not Available.”

Missing data can happen for many reasons:

  • A survey participant skipped a question
  • A sensor failed to record a reading
  • A patient dropped out of a clinical trial

Why does this matter? Because if you ignore missing data without thinking about it, your results may be wrong. Some methods in R quietly remove missing values — you need to understand when that is happening and whether it is safe to do so.

Note🧠 Analogy

Think of a missing value like a blank in an exam paper. If you calculate the average score for the class and some students left questions blank, you need to decide: do you treat the blank as zero, skip that question entirely, or estimate what the student might have written? Each choice leads to a different average — and some choices are more honest than others.

2.2 R Functions for Missing Data

2.2.1 Basic Functions

These functions are useful when you have only a few missing values and you know there is no systematic pattern to which values are missing.

Code 
is.na(x)             # Returns TRUE/FALSE for each value — is it missing?
which(is.na(x))      # Tells you the *position* (index number) of each NA
na.rm = TRUE         # Used inside functions like mean() to skip NAs
complete.cases(x)    # Returns TRUE for rows where all values are present
na.omit(x)           # Removes all rows that contain at least one NA
Important⭐ Professor’s Note on These Functions

“These functions are useful when you only have a few missing data values. If your dataset is small enough, and you know that you only have a few missing values, or that you know that there’s no missing data patterns within your dataset, then these functions can be useful.”

— Paula Harrison

2.2.2 Removing Rows with Conditions

Code 
# Remove rows where a specific column has NA
df <- df[!is.na(df$specific_column), ]

# Remove rows where EITHER col1 OR col2 has NA
df <- df[!(is.na(df$col1) | is.na(df$col2)), ]
Warning⚠️ Common Mistake

Beginners often just run na.omit(df) without checking what they are losing. If a column has 55% of its values missing, removing all rows with any NA in that column could dramatically shrink your dataset. Always check how many complete cases remain after removal.

Professor’s note: “By eliminating Column 3 [which had 55% missing], we went from only 5 subjects with complete data to 12. So just by removing one column, we changed the completeness of the dataset — but we also changed the sample size. You would have to investigate the extent to which data is missing and make sure the data you have left is enough to be a truly representative sample.”

2.3 The Sleep Dataset

The examples this week use the sleep dataset — a classic study of sleep patterns across 39 mammal species (62 observations, 10 variables). This dataset is built into the VIM package and is a good example because it has a realistic spread of missing values.

Variable Type Description
BodyWgt Continuous Body weight (kg)
BrainWgt Continuous Brain weight (kg)
NonD Continuous Non-dream sleep (hours/day)
Dream Continuous Dream sleep (hours/day)
Sleep Continuous Total sleep = NonD + Dream (hours/day)
Span Continuous Maximum life span (years)
Gest Continuous Gestation time (days)
Pred Categorical Predation index (danger from predators)
Exp Categorical Sleep exposure index
Danger Categorical Overall danger (1 = least, 5 = most)

Reference: Allison & Chichetti (1976), Science 194(4266), 732–734.

Note💡 Why NonD + Dream = Sleep?

The Sleep variable is just the sum of NonD and Dream. This means if either of those two is missing, then Sleep will also be missing. This is an example of structured missing data — missingness in one variable causes missingness in another. The combination plot from aggr() reveals this pattern clearly.

2.4 Summarising Data with summarytools

The summarytools package provides a quick, readable summary of your entire dataset. It shows you the mean, standard deviation, min, max, distinct values, and number of valid (non-missing) observations for each column.

Code 
library(summarytools)
dfSummary(sleep)

This is a good first step in any analysis. If a column has fewer valid values than the total number of rows, that tells you there are missing values in that column.

Professor’s note: “It’s a useful package if you want to just quickly summarise your dataset — useful for seeing if it’s skewed or normal, how many distinct values there are, the mean, standard deviation, median, max, IQR, coefficient of variance.”

2.5 Visualising Missing Data (VIM Package)

The VIM package provides specialised plots for exploring missing data. The key colour convention across all VIM plots is:

  • 🔴 Red = Missing
  • 🔵 Blue = Complete

2.5.1 aggr() — Count and Show Patterns of Missingness

Code 
library(VIM)
aggr(sleep, numbers = TRUE, prop = FALSE)

This produces two plots side by side:

Left plot (bar chart): How many values are missing in each variable. In the sleep data, NonD had ~14 missing values — the most of any variable.

Right plot (combination matrix): Which variables tend to be missing at the same time. For example, the combination matrix showed that rows 31 and 62 both had NonD, Dream, and Sleep missing simultaneously. This makes sense because Sleep = NonD + Dream.

Important⭐ Always Double-Check Your Plots

Professor’s note: “Don’t assume that the results are correct. Just if you can, try to test to see if the results are correct. So I’ve taken this combination showing NonD, Dream and Sleep missing together — we know we have two rows where those three variables are missing at the same time, and I went back to the dataset and had a look to make sure that was correct.”

Always verify surprising patterns by inspecting the actual rows in your data (e.g., sleep[c(31, 62), ]).

2.5.2 barMiss() — Missingness by a Categorical Variable

Use this when you want to explore whether missingness in one variable is related to the values of a categorical variable.

Code 
barMiss(sleep[, c("Exp", "Sleep")])

How to read this plot: The x-axis shows categories of Exp (the exposure index). Each bar shows the total count of Sleep observations in that category. Red sections show how many Sleep values are missing within that category.

Findings for sleep data: - Sleep had 4 missing values in total - 1 missing when Exp = 1 - 3 missing when Exp = 5 - Exp itself had no missing values

The last column (“missing”) shows how many Exp observations were missing — in this case, zero.

2.5.3 histMiss() — Missingness by a Continuous Variable

This is the same idea as barMiss(), but for continuous variables, where a histogram is more appropriate than a bar chart.

Code 
histMiss(sleep[, c("Span", "Sleep")])

Findings for sleep data: - Sleep had 4 missing values - 2 occurred when Span was between 10 and 20 years - 2 occurred when Span was between 20 and 30 years - Span itself had 4 missing values (shown in the separate “missing” bar on the right)

2.5.4 marginplot() — Comparing Distributions With and Without Missingness

This is the most informative VIM plot. It shows a scatterplot of two variables, with extra boxplots added in the margins.

Code 
marginplot(sleep[, c("Span", "Sleep")])

How to read this plot:

Element What it shows
🔴 Red dots (bottom margin) Number of missing observations in Span
🔴 Red dots (left margin) Number of missing observations in Sleep
🔴 Red boxplot (left margin) Distribution of Sleep where Span is missing
🔵 Blue boxplot (left margin) Distribution of Sleep where Span is observed
🔴 Red boxplot (bottom margin) Distribution of Span where Sleep is missing
🔵 Blue boxplot (bottom margin) Distribution of Span where Sleep is observed
Centre scatterplot All observations with complete data for both variables

Key question to ask: Are the red and blue boxplots similar or different?

  • If similar → the missingness of Span is not related to the values of Sleep. This is consistent with Missing At Random (MAR).
  • If different → the missingness pattern may be informative, which could bias your analysis if you simply delete the missing rows.

Professor’s note: “Blue is not missing, red is missing — and in the centre is just your standard scatterplot.”

2.6 Dealing with Missing Data

2.6.1 Option 1: Complete Case Analysis (Deleting Missing Rows)

The simplest approach is to remove any row that contains a missing value and perform your analysis on only the complete rows.

In R’s lm() function, this is the default behaviour — it automatically uses only complete cases.

Code 
# These two models give identical results:
lm(Sleep ~ BodyWgt, data = sleep)
lm(Sleep ~ BodyWgt, data = na.omit(sleep))

Other related functions:

Code 
na.fail()     # Throws an error if any NAs are present — forces you to address the issue
na.exclude()  # Like na.omit() but keeps empty slots for NA predictions
Warning⚠️ When is Complete Case Analysis Acceptable?

Only when:

  1. The amount of missing data is small
  2. There is no systematic pattern to which cases are missing (i.e., the data is Missing Completely At Random, MCAR)

If certain types of cases are more likely to be missing (e.g., sicker patients dropping out of a study), removing them introduces bias — your remaining sample no longer represents the full population.

Professor’s note: “When you ask ‘is it okay just to remove the data?’ — it depends on your analysis, and if there is no missing data patterns. But you can see with this dataset how the complete cases improve by just removing one column. You would have to investigate the extent to which data is missing.”

2.6.2 Option 2: Imputation with irmi() — Iterative Robust Model-Based Imputation

Imputation means filling in missing values with estimated values — rather than discarding incomplete rows.

The irmi() function in the VIM package does this through a series of regression models.

Code 
library(VIM)
imputed_sleep <- irmi(sleep)

How irmi() works (step by step):

  1. Pick one variable (e.g., NonD) to be the response (outcome) variable.
  2. Use all other variables as predictors in a regression model.
  3. Where NonD is missing, use the predicted value from the regression as the imputed value.
  4. Move on to the next variable with missing values, and repeat.
  5. Keep cycling through all variables until the imputed values stabilise (i.e., stop changing much between iterations).

“Irmi — that is, Iterative Robust Model-Based Imputation. It basically runs iterative regression analysis in which each iteration uses one variable as the response variable and the remaining variables as the explanatory variables.” — Paula Harrison

After imputation, compare your models:

Code 
# Before imputation — uses complete cases only (42 of 62 rows)
model_before <- lm(Sleep ~ BodyWgt, data = sleep)

# After imputation — uses all 62 rows
model_after  <- lm(Sleep ~ BodyWgt, data = imputed_sleep)

summary(model_before)
summary(model_after)

Expected findings: The intercept will be slightly larger for the imputed model, and the slope slightly less negative — because the imputed model now uses all 62 rows, not just the 42 complete cases.

2.6.3 Option 3: Multiple Imputation with MICE

The MICE package (Multiple Imputation by Chained Equations) is more sophisticated. Instead of creating one imputed dataset, it creates multiple datasets (e.g., 5), fits a model to each one separately, and then pools (averages) the results.

Code 
library(mice)

# Step 1: Create 5 imputed datasets using predictive mean matching
imputed <- mice(sleep, m = 5, method = "pmm")

# Step 2: Fit a linear model to each of the 5 datasets
models  <- with(imputed, lm(Sleep ~ BodyWgt))

# Step 3: Pool (average) the results across all 5 models
pooled  <- pool(models)
summary(pooled)

What is predictive mean matching (PMM)?

PMM works by: 1. Predicting a value for the missing observation using a regression model. 2. Finding a small set of real observed values that are closest to that prediction. 3. Randomly choosing one of those real values as the imputed value.

This means imputed values are always real values from your dataset — they will never be unrealistically extreme.

Important⭐ Why Are MICE Standard Errors Always Larger?

Professor’s note: “The standard error is larger for the pooled model — it’s always going to be larger — because it’s including the extra uncertainty that we have created in our dataset due to the imputation values.”

When we impute missing data, we are making an educated guess. The pooled standard errors are larger than those from a single imputed model because they account for the uncertainty introduced by not knowing the true values. This is more statistically honest than pretending our imputed values are perfectly correct.

Note💡 MICE vs irmi() — When to Use Which?
irmi() MICE
Number of imputed datasets 1 Multiple (default = 5)
Accounts for imputation uncertainty? No Yes
Pooled standard errors Not available Available
Complexity Simpler More complex
Best used when Quick analysis, small data Publication-quality results, large missing proportion

3 Part 2: Power Analysis

3.1 What Is Statistical Power?

When you run a hypothesis test, there are four possible outcomes — two correct decisions and two types of error:

H₀ is True H₀ is False
Reject H₀ ❌ Type I error (probability = α) ✅ Correct! (probability = power = 1 − β)
Fail to reject H₀ ✅ Correct! (probability = 1 − α) ❌ Type II error (probability = β)

Statistical power is the probability that your test will correctly detect a real effect when one truly exists:

\[\text{Power} = 1 - \beta = P(\text{reject } H_0 \mid H_0 \text{ is false})\]

In plain English: if there is a real difference, how likely is your test to find it?

Note🧠 Analogy

Think of power like the sensitivity of a metal detector on a beach. A low-power detector might only find a large coin buried 2 cm deep. A high-power detector will find a small earring buried 30 cm deep. If you want to detect small or subtle effects, you need a high-power test — and that usually means a larger sample.

Professor’s note: “The statistical power of the hypothesis test is defined to be the probability of rejecting the null hypothesis when it is false. So 1 minus beta. So basically, to get the correct result.”

3.2 The Four Interconnected Quantities

Power analysis is built on the relationship between four quantities. If you know any three, you can calculate the fourth:

Quantity Symbol Typical default
Sample size n Chosen based on budget/feasibility
Effect size δ The minimum meaningful difference
Significance level (Type I error rate) α 0.05
Power (1 − Type II error rate) 1 − β ≥ 0.80
Important⭐ You Cannot Know the True Effect Size

Professor’s note: “In reality, it’s not possible to know the true effect size. Because we don’t generally know the true population mean. So we don’t know what the true difference is that we’re looking for. So what we are going to choose instead is to have a minimum effect size — what is the minimum difference in means that we’re willing to accept as a practical difference?”

In practice, you define the minimum effect size of practical importance — the smallest difference that would actually matter for your application. This comes from prior research, expert knowledge, or clinical guidelines.

3.3 Clinical vs Statistical Significance

Statistical significance means your test produced a p-value below α — the result is unlikely to be due to random chance.

Clinical (or practical) significance means the size of the effect is large enough to matter in the real world.

Note💡 The Fethney (2010) Example

This paper (available on Canvas) gives a real example of defining a minimum effect size.

Premature infants were randomly assigned to one of two treatment groups across four weight-gain interventions. Group A received the intervention; Group B received standard treatment. Weights were compared using the Mann-Whitney U-test.

The researchers decided in advance that a weight gain difference of 500 grams or more would be clinically significant. A negative lower confidence interval limit would indicate weight loss.

Professor’s note: “They have decided, based on previous research and knowledge, that the minimum difference that they want is 500 grams. So 500 grams is the minimum that we need in order for us to reject the null hypothesis.”

Key lesson: Before collecting data, you should define what size of difference you actually care about detecting.

3.4 Key Formulas

3.4.1 Z-Test Statistic (Population Standard Deviation Known)

\[Z_{stat} = \frac{\bar{x} - \mu_0}{\sigma / \sqrt{n}}\]

  • \(\bar{x}\) = sample mean
  • \(\mu_0\) = the null hypothesis value of the population mean
  • \(\sigma\) = known population standard deviation
  • \(n\) = sample size

3.4.2 Critical Sample Mean (Threshold for Rejecting H₀)

\[\bar{X}_{crit} = \mu_0 + \frac{\sigma}{\sqrt{n}} \times Z_{crit}\]

If the sample mean exceeds \(\bar{X}_{crit}\), we reject H₀.

3.4.3 Calculating β (Probability of Type II Error)

Once you have \(\bar{X}_{crit}\), calculate the probability that the sample mean falls below it under the alternative hypothesis (i.e., assuming the true mean is \(\mu_A\)):

\[\beta = P\left(\bar{X} < \bar{X}_{crit} \mid \mu = \mu_A\right) = P\left(Z < \frac{\bar{X}_{crit} - \mu_A}{\sigma / \sqrt{n}}\right)\]

\[\text{Power} = 1 - \beta\]

3.4.4 Sample Size Formula — Known σ (from Confidence Interval Margin of Error)

\[ME = Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}} \quad \Rightarrow \quad n = \left(\frac{Z_{\alpha/2} \times \sigma}{ME}\right)^2\]

Warning⚠️ Always Round Sample Size Up

If your formula gives n = 77.8, your sample size is 78, not 77. You cannot sample a fraction of a person (or resistor, or container). Always round up to the next whole number.

Professor’s note: “You can’t sample 77.8 units, so what is the sample size that you need? 78 — you would always go one up, yeah. Just round up.”

3.4.5 Sample Size Formula — Unknown σ (t-Distribution)

When the population standard deviation is unknown, use the sample standard deviation \(s\) and the t-distribution:

\[n = \left(\frac{t_{\alpha/2,\, n-1} \times s}{ME}\right)^2\]

Note: this requires iteration because \(t_{n-1}\) depends on \(n\) — start with an initial estimate, then recalculate.

3.5 Worked Example — Chemical Company

3.5.1 Problem Setup

A chemical company claims their product contains on average μ₀ = 4.0 ml of caustic material per litre. If the true mean exceeds 4.0, the product is dangerous.

A government inspector tests n = 30 containers and finds: - Sample mean: \(\bar{x} = 4.3\) ml - Known population standard deviation: \(\sigma = 1.3\) ml

3.5.2 Step 1: State the Hypotheses

The product is only dangerous if it contains more than 4.0 ml — so this is a one-tailed test (right-tailed):

\[H_0: \mu = 4.0 \qquad H_A: \mu > 4.0\]

3.5.3 Step 2: Identify the Error Types in Context

  • Type I error (α): Concluding the mean is greater than 4.0 ml when it actually is not. The inspector falsely accuses the company.
  • Type II error (β): Concluding the mean is not greater than 4.0 ml when it actually is. The inspector misses a genuine danger.

Professor’s note: “You can never say that the null hypothesis is true — it’s just that we haven’t disproved the null hypothesis. And we can’t say that the alternative hypothesis is true either. We can just say that it’s more likely.”

3.5.4 Step 3: Conduct the Hypothesis Test (α = 0.05)

\[Z_{stat} = \frac{4.3 - 4.0}{1.3 / \sqrt{30}} = \frac{0.3}{0.2374} = 1.264\]

For a one-tailed test at α = 0.05, the critical value is \(Z_{crit} = 1.645\).

Since \(Z_{stat} = 1.264 < 1.645\), we fail to reject H₀.

The p-value is 0.103 > 0.05, confirming this decision.

Conclusion: The data does not provide sufficient evidence to conclude that the mean level of caustic material exceeds 4 ml per litre.

The 95% lower bound is 3.910 — we are 95% confident that the true mean is greater than 3.910 ml.

Important⭐ Professor’s Decision Rule (Critical Value Method)

“If the test statistic is greater than the critical value, we reject. If it’s less than, we fail to reject the critical value. So if the Z score is less than 1.645, which it is in this case, then our decision is that we fail to reject the null hypothesis.”

Summary: - \(Z_{stat} > Z_{crit}\) → Reject H₀ - \(Z_{stat} < Z_{crit}\) → Fail to reject H₀

3.5.5 Step 4: Calculate the Power of the Test

The test did not find a significant result — but was the sample large enough to detect a real difference?

Suppose a difference of 0.35 ml/litre is practically important — i.e., we want to detect if the true mean is \(\mu_A = 4.35\).

Find the critical sample mean (rejection threshold):

\[\bar{X}_{crit} = 4.0 + \frac{1.3}{\sqrt{30}} \times 1.645 = 4.0 + 0.39 = 4.39\]

We only reject H₀ if the sample mean exceeds 4.39 ml.

Calculate β (probability of Type II error under μ_A = 4.35):

\[\beta = P(\bar{X} < 4.39 \mid \mu = 4.35) = P\left(Z < \frac{4.39 - 4.35}{1.3/\sqrt{30}}\right) = P(Z < 0.17) = 0.5675\]

Power:

\[\text{Power} = 1 - 0.5675 = \mathbf{0.4325} \approx 43.25\%\]

Interpretation: There is only a 43% chance of correctly detecting a 0.35 ml difference with n = 30. This is quite low — the test is underpowered. We would ideally want power ≥ 0.80.

Important⭐ What Low Power Means

Professor’s note: “There’s a greater chance that we’re going to commit a Type 2 error. The probability is higher for committing a Type 2 error over a Type 1 error. And that could be because the sample size is small. Maybe it’s to do with the difference that we are looking to see — the 0.35 could be to do with that.”

A test result of “fail to reject” is not the same as proof that there is no effect. It might just mean your sample was too small to detect the effect you were looking for.

3.6 Sample Size from Margin of Error

Power analysis connects directly to confidence interval precision. The margin of error (ME) in a confidence interval is:

\[ME = Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}}\]

Rearranging to solve for the required sample size:

\[n = \left(\frac{Z_{\alpha/2} \times \sigma}{ME}\right)^2\]

3.6.1 Worked Example: Resistor Voltage

An analyst wants to estimate the mean voltage of resistors. The population standard deviation is σ = 22.5 volts. What sample size is needed to achieve a margin of error of 5 volts at a 95% confidence level (two-sided)?

\[n = \left(\frac{1.96 \times 22.5}{5}\right)^2 = \left(\frac{44.1}{5}\right)^2 = (8.82)^2 = 77.8 \approx \mathbf{78}\]

5 Practice Questions with Answers

Important⭐ Practice Question 1

Recalculate the power of the caustic material test if the minimum detectable increase is 0.1 ml instead of 0.35 ml.

Answer:

  1. \(\bar{X}_{crit} = 4.39\) (unchanged — same H₀, n, σ, α)
  2. \(\mu_A = 4.0 + 0.1 = 4.10\)
  3. \(Z = (4.39 - 4.10) / (1.3/\sqrt{30}) = 0.29 / 0.2374 = 1.22\)
  4. \(\beta = P(Z < 1.22) = 0.8888\)
  5. Power \(= 1 - 0.8888 = \mathbf{0.1112}\) ≈ 11.1%

This is even lower than before — because we are trying to detect a smaller difference (0.1 ml vs 0.35 ml) with the same small sample.

Important⭐ Practice Question 2

An analyst wants to estimate the mean voltage of resistors (σ = 22.5 volts). What sample size achieves a margin of error of 5 volts at 95% confidence for a two-sided CI?

Answer: n = (1.96 × 22.5 / 5)² = 77.8 → rounded up to n = 78

# Part 1: Missing Data

## What Is Missing Data?

In real-world datasets, it is very common to find that some values are not recorded. These are called **missing values**. In R, missing values are stored as `NA` — short for "Not Available."

Missing data can happen for many reasons:

- A survey participant skipped a question
- A sensor failed to record a reading
- A patient dropped out of a clinical trial

**Why does this matter?** Because if you ignore missing data without thinking about it, your results may be wrong. Some methods in R quietly remove missing values — you need to understand when that is happening and whether it is safe to do so.

::: {.callout-note}
## 🧠 Analogy

Think of a missing value like a blank in an exam paper. If you calculate the average score for the class and some students left questions blank, you need to decide: do you treat the blank as zero, skip that question entirely, or estimate what the student might have written? Each choice leads to a different average — and some choices are more honest than others.
:::

---

## R Functions for Missing Data

### Basic Functions

These functions are useful when you have **only a few missing values** and you know there is no systematic pattern to which values are missing.

```{r}
#| eval: false

is.na(x)             # Returns TRUE/FALSE for each value — is it missing?
which(is.na(x))      # Tells you the *position* (index number) of each NA
na.rm = TRUE         # Used inside functions like mean() to skip NAs
complete.cases(x)    # Returns TRUE for rows where all values are present
na.omit(x)           # Removes all rows that contain at least one NA
```

::: {.callout-important}
## ⭐ Professor's Note on These Functions

*"These functions are useful when you only have a few missing data values. If your dataset is small enough, and you know that you only have a few missing values, or that you know that there's no missing data patterns within your dataset, then these functions can be useful."*

— Paula Harrison
:::

### Removing Rows with Conditions

```{r}
#| eval: false

# Remove rows where a specific column has NA
df <- df[!is.na(df$specific_column), ]

# Remove rows where EITHER col1 OR col2 has NA
df <- df[!(is.na(df$col1) | is.na(df$col2)), ]
```

::: {.callout-warning}
## ⚠️ Common Mistake

Beginners often just run `na.omit(df)` without checking what they are losing. If a column has 55% of its values missing, removing all rows with any NA in that column could dramatically shrink your dataset. Always check how many complete cases remain after removal.

*Professor's note: "By eliminating Column 3 [which had 55% missing], we went from only 5 subjects with complete data to 12. So just by removing one column, we changed the completeness of the dataset — but we also changed the sample size. You would have to investigate the extent to which data is missing and make sure the data you have left is enough to be a truly representative sample."*
:::

## Dealing with Missing Data

### Option 1: Complete Case Analysis (Deleting Missing Rows)

The simplest approach is to remove any row that contains a missing value and perform your analysis on only the complete rows.

In R's `lm()` function, this is the **default behaviour** — it automatically uses only complete cases.

```{r}
#| eval: false

# These two models give identical results:
lm(Sleep ~ BodyWgt, data = sleep)
lm(Sleep ~ BodyWgt, data = na.omit(sleep))
```

**Other related functions:**

```{r}
#| eval: false

na.fail()     # Throws an error if any NAs are present — forces you to address the issue
na.exclude()  # Like na.omit() but keeps empty slots for NA predictions
```

::: {.callout-warning}
## ⚠️ When is Complete Case Analysis Acceptable?

Only when:

1. The amount of missing data is **small**
2. There is **no systematic pattern** to which cases are missing (i.e., the data is Missing Completely At Random, MCAR)

If certain types of cases are more likely to be missing (e.g., sicker patients dropping out of a study), removing them introduces **bias** — your remaining sample no longer represents the full population.

*Professor's note: "When you ask 'is it okay just to remove the data?' — it depends on your analysis, and if there is no missing data patterns. But you can see with this dataset how the complete cases improve by just removing one column. You would have to investigate the extent to which data is missing."*
:::

---

### Option 2: Imputation with `irmi()` — Iterative Robust Model-Based Imputation

**Imputation** means filling in missing values with estimated values — rather than discarding incomplete rows.

The `irmi()` function in the **VIM package** does this through a series of regression models.

```{r}
#| eval: false

library(VIM)
imputed_sleep <- irmi(sleep)
```

**How `irmi()` works (step by step):**

1. Pick one variable (e.g., `NonD`) to be the **response** (outcome) variable.
2. Use all other variables as **predictors** in a regression model.
3. Where `NonD` is missing, use the predicted value from the regression as the imputed value.
4. Move on to the next variable with missing values, and repeat.
5. Keep cycling through all variables until the imputed values stabilise (i.e., stop changing much between iterations).

*"Irmi — that is, Iterative Robust Model-Based Imputation. It basically runs iterative regression analysis in which each iteration uses one variable as the response variable and the remaining variables as the explanatory variables."*
— Paula Harrison

**After imputation, compare your models:**

```{r}
#| eval: false

# Before imputation — uses complete cases only (42 of 62 rows)
model_before <- lm(Sleep ~ BodyWgt, data = sleep)

# After imputation — uses all 62 rows
model_after  <- lm(Sleep ~ BodyWgt, data = imputed_sleep)

summary(model_before)
summary(model_after)
```

**Expected findings:** The intercept will be slightly larger for the imputed model, and the slope slightly less negative — because the imputed model now uses all 62 rows, not just the 42 complete cases.

---

### Option 3: Multiple Imputation with MICE

The **MICE** package (Multiple Imputation by Chained Equations) is more sophisticated. Instead of creating one imputed dataset, it creates **multiple** datasets (e.g., 5), fits a model to each one separately, and then **pools** (averages) the results.

```{r}
#| eval: false

library(mice)

# Step 1: Create 5 imputed datasets using predictive mean matching
imputed <- mice(sleep, m = 5, method = "pmm")

# Step 2: Fit a linear model to each of the 5 datasets
models  <- with(imputed, lm(Sleep ~ BodyWgt))

# Step 3: Pool (average) the results across all 5 models
pooled  <- pool(models)
summary(pooled)
```

**What is predictive mean matching (PMM)?**

PMM works by:
1. Predicting a value for the missing observation using a regression model.
2. Finding a small set of **real observed values** that are closest to that prediction.
3. Randomly choosing one of those real values as the imputed value.

This means imputed values are always **real values from your dataset** — they will never be unrealistically extreme.

::: {.callout-important}
## ⭐ Why Are MICE Standard Errors Always Larger?

*Professor's note: "The standard error is larger for the pooled model — it's always going to be larger — because it's including the extra uncertainty that we have created in our dataset due to the imputation values."*

When we impute missing data, we are making an educated guess. The pooled standard errors are **larger** than those from a single imputed model because they account for the uncertainty introduced by not knowing the true values. This is more statistically **honest** than pretending our imputed values are perfectly correct.
:::

::: {.callout-note}
## 💡 MICE vs irmi() — When to Use Which?

| | `irmi()` | MICE |
|---|---|---|
| Number of imputed datasets | 1 | Multiple (default = 5) |
| Accounts for imputation uncertainty? | No | Yes |
| Pooled standard errors | Not available | Available |
| Complexity | Simpler | More complex |
| Best used when | Quick analysis, small data | Publication-quality results, large missing proportion |
:::

# Part 2: Power Analysis

## What Is Statistical Power?

When you run a hypothesis test, there are four possible outcomes — two correct decisions and two types of error:

| | H₀ is **True** | H₀ is **False** |
|---|---|---|
| **Reject H₀** | ❌ Type I error (probability = α) | ✅ Correct! (probability = **power** = 1 − β) |
| **Fail to reject H₀** | ✅ Correct! (probability = 1 − α) | ❌ Type II error (probability = β) |

**Statistical power** is the probability that your test will **correctly detect a real effect** when one truly exists:

$$\text{Power} = 1 - \beta = P(\text{reject } H_0 \mid H_0 \text{ is false})$$

In plain English: if there is a real difference, how likely is your test to find it?

::: {.callout-note}
## 🧠 Analogy

Think of power like the sensitivity of a metal detector on a beach. A low-power detector might only find a large coin buried 2 cm deep. A high-power detector will find a small earring buried 30 cm deep. If you want to detect small or subtle effects, you need a high-power test — and that usually means a larger sample.
:::

*Professor's note: "The statistical power of the hypothesis test is defined to be the probability of rejecting the null hypothesis when it is false. So 1 minus beta. So basically, to get the correct result."*

---

## The Four Interconnected Quantities

Power analysis is built on the relationship between four quantities. **If you know any three, you can calculate the fourth:**

| Quantity | Symbol | Typical default |
|---|---|---|
| Sample size | n | Chosen based on budget/feasibility |
| Effect size | δ | The minimum meaningful difference |
| Significance level (Type I error rate) | α | 0.05 |
| Power (1 − Type II error rate) | 1 − β | ≥ 0.80 |

::: {.callout-important}
## ⭐ You Cannot Know the True Effect Size

*Professor's note: "In reality, it's not possible to know the true effect size. Because we don't generally know the true population mean. So we don't know what the true difference is that we're looking for. So what we are going to choose instead is to have a minimum effect size — what is the minimum difference in means that we're willing to accept as a practical difference?"*

In practice, you define the **minimum effect size of practical importance** — the smallest difference that would actually matter for your application. This comes from prior research, expert knowledge, or clinical guidelines.
:::

---

## Clinical vs Statistical Significance

**Statistical significance** means your test produced a p-value below α — the result is unlikely to be due to random chance.

**Clinical (or practical) significance** means the size of the effect is large enough to matter in the real world.

::: {.callout-note}
## 💡 The Fethney (2010) Example

This paper (available on Canvas) gives a real example of defining a minimum effect size.

Premature infants were randomly assigned to one of two treatment groups across four weight-gain interventions. Group A received the intervention; Group B received standard treatment. Weights were compared using the Mann-Whitney U-test.

The researchers decided in advance that **a weight gain difference of 500 grams or more** would be clinically significant. A negative lower confidence interval limit would indicate weight loss.

*Professor's note: "They have decided, based on previous research and knowledge, that the minimum difference that they want is 500 grams. So 500 grams is the minimum that we need in order for us to reject the null hypothesis."*

**Key lesson:** Before collecting data, you should define what size of difference you actually care about detecting.
:::

---

## Key Formulas

### Z-Test Statistic (Population Standard Deviation Known)

$$Z_{stat} = \frac{\bar{x} - \mu_0}{\sigma / \sqrt{n}}$$

- $\bar{x}$ = sample mean
- $\mu_0$ = the null hypothesis value of the population mean
- $\sigma$ = known population standard deviation
- $n$ = sample size

### Critical Sample Mean (Threshold for Rejecting H₀)

$$\bar{X}_{crit} = \mu_0 + \frac{\sigma}{\sqrt{n}} \times Z_{crit}$$

If the sample mean exceeds $\bar{X}_{crit}$, we reject H₀.

### Calculating β (Probability of Type II Error)

Once you have $\bar{X}_{crit}$, calculate the probability that the sample mean falls below it **under the alternative hypothesis** (i.e., assuming the true mean is $\mu_A$):

$$\beta = P\left(\bar{X} < \bar{X}_{crit} \mid \mu = \mu_A\right) = P\left(Z < \frac{\bar{X}_{crit} - \mu_A}{\sigma / \sqrt{n}}\right)$$

$$\text{Power} = 1 - \beta$$

### Sample Size Formula — Known σ (from Confidence Interval Margin of Error)

$$ME = Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}} \quad \Rightarrow \quad n = \left(\frac{Z_{\alpha/2} \times \sigma}{ME}\right)^2$$

::: {.callout-warning}
## ⚠️ Always Round Sample Size **Up**

If your formula gives n = 77.8, your sample size is **78**, not 77. You cannot sample a fraction of a person (or resistor, or container). Always round up to the next whole number.

*Professor's note: "You can't sample 77.8 units, so what is the sample size that you need? 78 — you would always go one up, yeah. Just round up."*
:::

### Sample Size Formula — Unknown σ (t-Distribution)

When the population standard deviation is unknown, use the sample standard deviation $s$ and the t-distribution:

$$n = \left(\frac{t_{\alpha/2,\, n-1} \times s}{ME}\right)^2$$

Note: this requires **iteration** because $t_{n-1}$ depends on $n$ — start with an initial estimate, then recalculate.

---

## Worked Example — Chemical Company

### Problem Setup

A chemical company claims their product contains on average **μ₀ = 4.0 ml** of caustic material per litre. If the true mean exceeds 4.0, the product is dangerous.

A government inspector tests **n = 30** containers and finds:
- Sample mean: $\bar{x} = 4.3$ ml
- Known population standard deviation: $\sigma = 1.3$ ml

### Step 1: State the Hypotheses

The product is only dangerous if it contains **more than** 4.0 ml — so this is a **one-tailed test** (right-tailed):

$$H_0: \mu = 4.0 \qquad H_A: \mu > 4.0$$

### Step 2: Identify the Error Types in Context

- **Type I error (α):** Concluding the mean is greater than 4.0 ml when it actually is not. The inspector falsely accuses the company.
- **Type II error (β):** Concluding the mean is not greater than 4.0 ml when it actually is. The inspector misses a genuine danger.

*Professor's note: "You can never say that the null hypothesis is true — it's just that we haven't disproved the null hypothesis. And we can't say that the alternative hypothesis is true either. We can just say that it's more likely."*

### Step 3: Conduct the Hypothesis Test (α = 0.05)

$$Z_{stat} = \frac{4.3 - 4.0}{1.3 / \sqrt{30}} = \frac{0.3}{0.2374} = 1.264$$

For a one-tailed test at α = 0.05, the critical value is $Z_{crit} = 1.645$.

Since $Z_{stat} = 1.264 < 1.645$, we **fail to reject H₀**.

The p-value is 0.103 > 0.05, confirming this decision.

**Conclusion:** The data does not provide sufficient evidence to conclude that the mean level of caustic material exceeds 4 ml per litre.

The 95% lower bound is 3.910 — we are 95% confident that the true mean is greater than 3.910 ml.

::: {.callout-important}
## ⭐ Professor's Decision Rule (Critical Value Method)

*"If the test statistic is greater than the critical value, we reject. If it's less than, we fail to reject the critical value. So if the Z score is less than 1.645, which it is in this case, then our decision is that we fail to reject the null hypothesis."*

**Summary:**
- $Z_{stat} > Z_{crit}$ → Reject H₀
- $Z_{stat} < Z_{crit}$ → Fail to reject H₀
:::

### Step 4: Calculate the Power of the Test

The test did not find a significant result — but **was the sample large enough to detect a real difference?**

Suppose a difference of **0.35 ml/litre** is practically important — i.e., we want to detect if the true mean is $\mu_A = 4.35$.

**Find the critical sample mean (rejection threshold):**

$$\bar{X}_{crit} = 4.0 + \frac{1.3}{\sqrt{30}} \times 1.645 = 4.0 + 0.39 = 4.39$$

We only reject H₀ if the sample mean exceeds **4.39 ml**.

**Calculate β (probability of Type II error under μ_A = 4.35):**

$$\beta = P(\bar{X} < 4.39 \mid \mu = 4.35) = P\left(Z < \frac{4.39 - 4.35}{1.3/\sqrt{30}}\right) = P(Z < 0.17) = 0.5675$$

**Power:**

$$\text{Power} = 1 - 0.5675 = \mathbf{0.4325} \approx 43.25\%$$

**Interpretation:** There is only a 43% chance of correctly detecting a 0.35 ml difference with n = 30. This is quite low — the test is **underpowered**. We would ideally want power ≥ 0.80.

::: {.callout-important}
## ⭐ What Low Power Means

*Professor's note: "There's a greater chance that we're going to commit a Type 2 error. The probability is higher for committing a Type 2 error over a Type 1 error. And that could be because the sample size is small. Maybe it's to do with the difference that we are looking to see — the 0.35 could be to do with that."*

A test result of "fail to reject" is **not the same as proof that there is no effect**. It might just mean your sample was too small to detect the effect you were looking for.
:::

---

## Sample Size from Margin of Error

Power analysis connects directly to confidence interval precision. The margin of error (ME) in a confidence interval is:

$$ME = Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}}$$

Rearranging to solve for the required sample size:

$$n = \left(\frac{Z_{\alpha/2} \times \sigma}{ME}\right)^2$$

### Worked Example: Resistor Voltage

An analyst wants to estimate the mean voltage of resistors. The population standard deviation is **σ = 22.5 volts**. What sample size is needed to achieve a margin of error of **5 volts** at a **95% confidence level** (two-sided)?

$$n = \left(\frac{1.96 \times 22.5}{5}\right)^2 = \left(\frac{44.1}{5}\right)^2 = (8.82)^2 = 77.8 \approx \mathbf{78}$$

# Practice Questions with Answers

::: {.callout-important}
## ⭐ Practice Question 1

Recalculate the power of the caustic material test if the minimum detectable increase is **0.1 ml** instead of 0.35 ml.

**Answer:**

1. $\bar{X}_{crit} = 4.39$ (unchanged — same H₀, n, σ, α)
2. $\mu_A = 4.0 + 0.1 = 4.10$
3. $Z = (4.39 - 4.10) / (1.3/\sqrt{30}) = 0.29 / 0.2374 = 1.22$
4. $\beta = P(Z < 1.22) = 0.8888$
5. Power $= 1 - 0.8888 = \mathbf{0.1112}$ ≈ 11.1%

This is even lower than before — because we are trying to detect a smaller difference (0.1 ml vs 0.35 ml) with the same small sample.
:::

::: {.callout-important}
## ⭐ Practice Question 2

An analyst wants to estimate the mean voltage of resistors (σ = 22.5 volts). What sample size achieves a margin of error of 5 volts at 95% confidence for a two-sided CI?

**Answer:** n = (1.96 × 22.5 / 5)² = 77.8 → rounded up to **n = 78**

# Summary Checklist

- ✅ Missing values in R are stored as `NA`. Use `is.na()`, `which()`, `complete.cases()`, and `na.omit()` to locate and remove them.
- ✅ Always visualise missing data before deciding how to handle it. The VIM functions `aggr()`, `barMiss()`, `histMiss()`, and `marginplot()` are your main tools.
- ✅ Red = missing, Blue = complete in all VIM plots.
- ✅ Complete case analysis (`na.omit`) is the default in `lm()`. It is only safe when data is missing completely at random and the proportion is small.
- ✅ `irmi()` does iterative regression-based imputation. It creates one complete dataset.
- ✅ MICE creates multiple imputed datasets, fits models to each, and pools results. The pooled standard errors are always larger — this is correct, not a mistake.
- ✅ **Statistical power = 1 − β** = probability of correctly detecting a real effect. Aim for ≥ 0.80.
- ✅ The four quantities in power analysis are: sample size (n), effect size, α, and power. Know three, find the fourth.
- ✅ In practice, you cannot know the true effect size. Define the **minimum practically important effect size** instead.
- ✅ Low power (like 43% in the caustic example) means the test may fail to detect a real, meaningful difference — usually because n is too small.
- ✅ Sample size formula (known σ): $n = (Z_{\alpha/2} \times \sigma / ME)^2$. Always round **up**.
- ✅ If σ is unknown, use the t-distribution formula: $n = (t_{\alpha/2,n-1} \times s / ME)^2$ (requires iteration).
- ✅ The final exam is **on campus**, not on Canvas.
