1 BLOCK 1 — Time Series Fundamentals

1.1 Concept Flash Cards

Read the question. Think of your answer. Then reveal with the toggle.

Q: What is a time series?

TipAnswer

A collection of observations recorded at regular, equally-spaced intervals over time. The equally-spaced condition is critical — if data is not equally spaced, it cannot be analysed as a time series.

Q: What are the four components of a time series?

TipAnswer
  1. Trend (T) — Long-term increase or decrease in the data
  2. Seasonal (S) — Repeating pattern that completes within one calendar year
  3. Cyclical (C) — Long irregular waves lasting more than 2 years (e.g., economic cycles)
  4. Random / Residual (R) — Unexplained noise left after removing other components

Q: What is the difference between a seasonal and a cyclical component?

TipAnswer
Seasonal Cyclical
Period Completes within 1 year Lasts more than 2 years
Pattern Regular and predictable Irregular, harder to predict
Example Christmas sales spike every December Economic recession every ~10 years

Q: When do you use an additive vs. multiplicative decomposition model?

TipAnswer
  • Additive → when the seasonal swings are constant in size over time (homogeneous seasonality). Formula: \(y_t = T_t + S_t + R_t\)
  • Multiplicative → when the seasonal swings grow larger as the trend increases (increasing variance). Formula: \(y_t = T_t \times S_t \times R_t\)

Visual test: Look at the plot. If the up-and-down swings get bigger over time → multiplicative.

Q: How can you convert a multiplicative model into an additive one?

TipAnswer

Take the logarithm of the data: \[\log(y_t) = \log(T_t) + \log(S_t) + \log(R_t)\] This stabilises the variance and turns multiplication into addition.

2 BLOCK 2 — Classical Models (Exponential Smoothing)

2.1 Concept Flash Cards

Q: What does “pure time series model” mean?

TipAnswer

A model that uses only the historical values of the variable itself to make forecasts — no external variables. Exponential smoothing methods are pure time series models. This makes them useful when you only have data for one variable.

Q: What does the smoothing parameter α (alpha) control in SES, and what happens at extreme values?

TipAnswer

α controls how much weight is given to recent vs. old observations:

α value Memory Effect
α → 1 Short memory Only the last observation matters (forecast = last value)
α → 0 Long memory All past values weighted equally (forecast = overall mean)
α = 0.83 (example) Short memory Most weight on recent values

R minimises the sum of squared errors to find the optimal α automatically.

Q: Name the three exponential smoothing methods and when to use each one.

TipAnswer
Method ETS Code Use when…
Simple Exponential Smoothing (SES) ETS(A,N,N) No trend, no seasonality
Holt’s (Double) ETS(A,A,N) Has trend, no seasonality
Holt-Winters (Triple) ETS(A,A,A) or ETS(A,A,M) Has trend AND seasonality

Additive Holt-Winters: seasonal effect is constant size → ETS(A,A,A)
Multiplicative Holt-Winters: seasonal effect grows → ETS(A,A,M)

Q: What does ETS stand for and what does each letter mean?

TipAnswer

Error · Trend · Seasonality

Each component can be: - N = None - A = Additive
- M = Multiplicative
- Ad = Additive damped (trend slows down over time)

Examples: ETS(A,N,N) = SES; ETS(A,A,A) = Holt-Winters Additive

Q: What is a “damped trend” and why might you prefer it?

TipAnswer

A damped trend is one that gradually flattens out as you project further into the future, rather than continuing at the same rate indefinitely. It is often more realistic because few trends continue forever at the same rate. In R: holt(data, damped = TRUE).

Q: What does the forecast look like for SES (no trend, no seasonality)?

TipAnswer

A flat horizontal line — all future forecast values are the same. This is because SES only estimates a level (weighted average of past), with no component to create upward/downward slope or seasonal waves. The confidence interval widens the further ahead you forecast.

3 BLOCK 3 — STL Decomposition

3.1 Concept Flash Cards

Q: What does STL stand for and what are its key advantages?

TipAnswer

Seasonal and Trend decomposition using Loess (Loess = local regression)

Advantages over basic decompose(): - No missing values at the beginning and end of the series - Robust to outliers — extreme values don’t distort the components - User can control the flexibility of the seasonal and trend windows

Disadvantage: Only works directly with additive models (take logs for multiplicative).

Q: What is the s.window = "periodic" argument in the stl() function?

TipAnswer

It tells R to assume the seasonal component is fixed (repeats identically each year). You can also give a number (e.g., s.window = 10) to allow the seasonality to change slowly over time. Using "periodic" is the default safe choice.

4 BLOCK 4 — Forecast Accuracy Metrics

4.1 Concept Flash Cards

Q: What do MAE, RMSE, and MAPE measure? Which penalises large errors more?

TipAnswer

Let \(e_t = y_t - \hat{y}_t\) (actual minus forecast):

Metric Formula Property
MAE \(\frac{1}{n}\sum|e_t|\) Average error size; treats all errors equally
RMSE \(\sqrt{\frac{1}{n}\sum e_t^2}\) Penalises large errors more (squaring amplifies big mistakes)
MAPE \(\frac{100}{n}\sum|\frac{e_t}{y_t}|\) Error as a %; good for comparing across different scales

RMSE penalises large errors most heavily.

Q: What is AIC and when is it used?

TipAnswer

Akaike Information Criterion — a measure of model quality that balances fit vs. complexity. It penalises models with more parameters (to prevent overfitting).

Rule: Lower AIC = better model. Used to compare models on the same dataset.

Also see: BIC (Bayesian IC) — similar but with a stronger complexity penalty.

Q: How do you split data for model evaluation in R?

TipAnswer 
train <- window(my_ts, end = c(2020, 12))   # training set
test  <- window(my_ts, start = c(2021, 1))  # test set

model <- ets(train)
accuracy(forecast(model, h = length(test)), test)

Compare RMSE and MAE on the test set — this is the honest measure of how well the model generalises.

5 BLOCK 5 — Stationarity & ARIMA

5.1 Concept Flash Cards

Q: What are the three conditions for a time series to be stationary?

TipAnswer
  1. Constant mean — the average does not drift up or down over time
  2. Constant variance — the spread of values stays the same throughout
  3. Constant covariance — if there’s a correlation at lag k, it is the same throughout the series (not just at one point in time)

Memory trick: Same mean, same spread, same memory — always, everywhere in the series.

Q: Why does ARIMA require stationary data?

TipAnswer

The statistical equations in ARIMA assume the data’s behaviour is stable and consistent throughout. If the mean is drifting, the model cannot reliably estimate what “normal” is. If variance is changing, predictions become unreliable. You must transform the data to be stationary first.

Q: What is differencing and when do you use it?

TipAnswer

Differencing means computing the change from one period to the next: \[\Delta y_t = y_t - y_{t-1}\]

Use it when your data has a non-constant mean (trend). The number of times you difference is the d in ARIMA(p,d,q).

  • d=1: removes a linear trend
  • d=2: removes a quadratic trend (rare)

For non-constant variance → use log transformation first.

Q: Explain ARIMA(p, d, q) — what does each letter mean?

TipAnswer
Letter Name Meaning
p AutoRegressive order How many past values influence today
d Integrated order How many times to difference the data
q Moving Average order How many past errors influence today

Example: ARIMA(1,1,1) = use last value + one differencing + last error.

Q: What is the difference between AR and MA components?

TipAnswer
AR (AutoRegressive) MA (Moving Average)
Depends on Past values of y Past forecast errors
Analogy “My commute today depends on my commute yesterday” “I adjust today based on how wrong I was yesterday”
Parameter φ (phi) θ (theta)

Q: How do you use ACF and PACF to identify the model?

TipAnswer
ACF pattern PACF pattern Model
Cuts off after lag q Decays slowly MA(q)
Decays slowly Cuts off after lag p AR(p)
Both decay slowly Both decay slowly ARMA(p,q)
All within bounds All within bounds White noise (no model needed)

“Cuts off” = drops suddenly inside the blue dashed lines
“Decays slowly” = gradually decreases bar by bar

Q: What is white noise, and why is it important?

TipAnswer

White noise is a series with: - Mean = 0
- Constant variance
- No autocorrelation (each observation is independent)

Why it matters: Your model’s residuals should look like white noise. If they do, the model has captured all the structure. If they don’t, the model is missing something.

6 BLOCK 6 — Residual Diagnostics

6.1 Concept Flash Cards

Q: What four plots do you use to check model residuals?

TipAnswer
Plot What you look for
Histogram Symmetric bell shape (normal distribution)
Boxplot Symmetric, median near zero, no extreme outliers
Q-Q plot Points follow the diagonal line
Residuals vs. Time Random scatter, no visible pattern or trend

Q: What does it mean if the Q-Q plot shows points curving off the line at both ends?

TipAnswer

It means the residuals have heavier tails than a normal distribution (more extreme values than expected). This is a sign that the model is not fully capturing all structure. You should note this in your report and potentially try a different model or transformation.

Q: What does standardising residuals mean and why do it?

TipAnswer

Standardising transforms residuals to have mean = 0 and standard deviation = 1: \[e_{std} = \frac{e_t - \bar{e}}{sd(e)}\]

Why: It lets you immediately see which values are unusual. Values beyond ±2 are potential outliers (outside 95% of a normal distribution). Values beyond ±3 are serious outliers.

7 BLOCK 7 — SARIMA & Model Identification

7.1 Concept Flash Cards

Q: How does SARIMA extend ARIMA?

TipAnswer

SARIMA adds a seasonal layer on top of ARIMA:

\[\text{SARIMA}(p,d,q)(P,D,Q)_m\]

  • Lowercase (p,d,q): handle regular (non-seasonal) autocorrelation
  • Uppercase (P,D,Q): handle seasonal autocorrelation at period m
  • m: the season length (12 = monthly, 4 = quarterly)

Example: SARIMA(1,1,1)(1,1,1)₁₂ for monthly data with annual seasonality.

Q: In auto.arima(), what does R do automatically?

TipAnswer

R automatically: 1. Tests for stationarity and chooses d 2. Searches over many combinations of p and q 3. Selects the model with the lowest AICc (corrected AIC) 4. Returns the best ARIMA or SARIMA model

It is a good starting point, but you should always check residuals and potentially compare manually specified alternatives.

8 BLOCK 8 — Tricky Concepts & Common Mistakes

8.1 Things students often get wrong

Warning

Common mistake 1: Confusing seasonal and cyclical components.
→ If it repeats within a year and is predictable: seasonal.
→ If it takes years and is irregular: cyclical.

Warning

Common mistake 2: Using a multiplicative model on data with zero or near-zero values.
→ Multiplicative models fail when values approach zero (you can’t multiply by zero). Use additive or log-transform first.

Warning

Common mistake 3: Thinking a low AIC always means the best model.
→ AIC and RMSE together tell a fuller story. A slightly higher AIC but much lower RMSE might still be preferred, especially if residuals look better.

Warning

Common mistake 4: Forgetting to check stationarity before ARIMA.
→ Always plot the series and run adf.test() before fitting ARIMA. Never skip this step.

Warning

Common mistake 5: Interpreting the Ljung-Box test backwards.
→ p > 0.05 = residuals ARE random = good model ✅
→ p < 0.05 = residuals have structure = model needs work ❌

**Q: What does STL stand for and what are its key advantages?**

::: {.callout-tip collapse="true" title="Answer"}
**Seasonal and Trend decomposition using Loess** (Loess = local regression)

**Advantages over basic `decompose()`:** - **No missing values** at the beginning and end of the series - **Robust** to outliers — extreme values don't distort the components - User can control the flexibility of the seasonal and trend windows

**Disadvantage:** Only works directly with additive models (take logs for multiplicative).
:::

------------------------------------------------------------------------

**Q: What is the `s.window = "periodic"` argument in the `stl()` function?**

::: {.callout-tip collapse="true" title="Answer"}
It tells R to assume the **seasonal component is fixed** (repeats identically each year). You can also give a number (e.g., `s.window = 10`) to allow the seasonality to change slowly over time. Using `"periodic"` is the default safe choice.
:::

# BLOCK 4 — Forecast Accuracy Metrics

## Concept Flash Cards

**Q: What do MAE, RMSE, and MAPE measure? Which penalises large errors more?**

::: {.callout-tip collapse="true" title="Answer"}
Let $e_t = y_t - \hat{y}_t$ (actual minus forecast):

| Metric | Formula | Property |
|------------------------|------------------------|------------------------|
| **MAE** | $\frac{1}{n}\sum|e_t|$ | Average error size; treats all errors equally |
| **RMSE** | $\sqrt{\frac{1}{n}\sum e_t^2}$ | **Penalises large errors more** (squaring amplifies big mistakes) |
| **MAPE** | $\frac{100}{n}\sum|\frac{e_t}{y_t}|$ | Error as a %; good for comparing across different scales |

RMSE penalises large errors most heavily.
:::

------------------------------------------------------------------------

**Q: What is AIC and when is it used?**

::: {.callout-tip collapse="true" title="Answer"}
**Akaike Information Criterion** — a measure of model quality that **balances fit vs. complexity**. It penalises models with more parameters (to prevent overfitting).

**Rule:** Lower AIC = better model. Used to compare models on the **same dataset**.

Also see: **BIC** (Bayesian IC) — similar but with a stronger complexity penalty.
:::

------------------------------------------------------------------------

**Q: How do you split data for model evaluation in R?**

::: {.callout-tip collapse="true" title="Answer"}
``` r
train <- window(my_ts, end = c(2020, 12))   # training set
test  <- window(my_ts, start = c(2021, 1))  # test set

model <- ets(train)
accuracy(forecast(model, h = length(test)), test)
```

Compare RMSE and MAE on the **test set** — this is the honest measure of how well the model generalises.
:::

# BLOCK 5 — Stationarity & ARIMA

## Concept Flash Cards

**Q: What are the three conditions for a time series to be stationary?**

::: {.callout-tip collapse="true" title="Answer"}
1.  **Constant mean** — the average does not drift up or down over time
2.  **Constant variance** — the spread of values stays the same throughout
3.  **Constant covariance** — if there's a correlation at lag k, it is the same throughout the series (not just at one point in time)

**Memory trick:** Same mean, same spread, same memory — always, everywhere in the series.
:::

------------------------------------------------------------------------

**Q: Why does ARIMA require stationary data?**

::: {.callout-tip collapse="true" title="Answer"}
The statistical equations in ARIMA assume the data's behaviour is **stable and consistent** throughout. If the mean is drifting, the model cannot reliably estimate what "normal" is. If variance is changing, predictions become unreliable. You must transform the data to be stationary first.
:::

------------------------------------------------------------------------

**Q: What is differencing and when do you use it?**

::: {.callout-tip collapse="true" title="Answer"}
Differencing means computing the **change** from one period to the next: $$\Delta y_t = y_t - y_{t-1}$$

Use it when your data has a **non-constant mean (trend)**. The number of times you difference is the **d** in ARIMA(p,d,q).

-   d=1: removes a linear trend\
-   d=2: removes a quadratic trend (rare)

For non-constant variance → use **log transformation** first.
:::

------------------------------------------------------------------------

**Q: Explain ARIMA(p, d, q) — what does each letter mean?**

::: {.callout-tip collapse="true" title="Answer"}
| Letter | Name                 | Meaning                                   |
|--------|----------------------|-------------------------------------------|
| **p**  | AutoRegressive order | How many past **values** influence today  |
| **d**  | Integrated order     | How many times to **difference** the data |
| **q**  | Moving Average order | How many past **errors** influence today  |

Example: ARIMA(1,1,1) = use last value + one differencing + last error.
:::

------------------------------------------------------------------------

**Q: What is the difference between AR and MA components?**

::: {.callout-tip collapse="true" title="Answer"}
|   | AR (AutoRegressive) | MA (Moving Average) |
|------------------------|------------------------|------------------------|
| Depends on | **Past values** of y | **Past forecast errors** |
| Analogy | "My commute today depends on my commute yesterday" | "I adjust today based on how wrong I was yesterday" |
| Parameter | φ (phi) | θ (theta) |
:::

------------------------------------------------------------------------

**Q: How do you use ACF and PACF to identify the model?**

::: {.callout-tip collapse="true" title="Answer"}
| ACF pattern | PACF pattern | Model |
|------------------------|------------------------|------------------------|
| Cuts off after lag **q** | Decays slowly | **MA(q)** |
| Decays slowly | Cuts off after lag **p** | **AR(p)** |
| Both decay slowly | Both decay slowly | **ARMA(p,q)** |
| All within bounds | All within bounds | **White noise** (no model needed) |

**"Cuts off"** = drops suddenly inside the blue dashed lines\
**"Decays slowly"** = gradually decreases bar by bar
:::

------------------------------------------------------------------------

**Q: What is white noise, and why is it important?**

::: {.callout-tip collapse="true" title="Answer"}
White noise is a series with: - Mean = 0\
- Constant variance\
- No autocorrelation (each observation is independent)

**Why it matters:** Your model's residuals should look like white noise. If they do, the model has captured all the structure. If they don't, the model is missing something.
:::

# BLOCK 6 — Residual Diagnostics

## Concept Flash Cards

**Q: What four plots do you use to check model residuals?**

::: {.callout-tip collapse="true" title="Answer"}
| Plot                   | What you look for                                |
|------------------------|--------------------------------------------------|
| **Histogram**          | Symmetric bell shape (normal distribution)       |
| **Boxplot**            | Symmetric, median near zero, no extreme outliers |
| **Q-Q plot**           | Points follow the diagonal line                  |
| **Residuals vs. Time** | Random scatter, no visible pattern or trend      |
:::

------------------------------------------------------------------------

**Q: What does it mean if the Q-Q plot shows points curving off the line at both ends?**

::: {.callout-tip collapse="true" title="Answer"}
It means the residuals have **heavier tails than a normal distribution** (more extreme values than expected). This is a sign that the model is not fully capturing all structure. You should note this in your report and potentially try a different model or transformation.
:::

------------------------------------------------------------------------

**Q: What does standardising residuals mean and why do it?**

::: {.callout-tip collapse="true" title="Answer"}
Standardising transforms residuals to have **mean = 0 and standard deviation = 1**: $$e_{std} = \frac{e_t - \bar{e}}{sd(e)}$$

Why: It lets you immediately see which values are unusual. Values **beyond ±2** are potential outliers (outside 95% of a normal distribution). Values **beyond ±3** are serious outliers.
:::

------------------------------------------------------------------------

# BLOCK 7 — SARIMA & Model Identification

## Concept Flash Cards

**Q: How does SARIMA extend ARIMA?**

::: {.callout-tip collapse="true" title="Answer"}
SARIMA adds a **seasonal layer** on top of ARIMA:

$$\text{SARIMA}(p,d,q)(P,D,Q)_m$$

-   Lowercase (p,d,q): handle regular (non-seasonal) autocorrelation
-   Uppercase (P,D,Q): handle seasonal autocorrelation at period **m**
-   **m**: the season length (12 = monthly, 4 = quarterly)

Example: SARIMA(1,1,1)(1,1,1)₁₂ for monthly data with annual seasonality.
:::

------------------------------------------------------------------------

**Q: In `auto.arima()`, what does R do automatically?**

::: {.callout-tip collapse="true" title="Answer"}
R automatically: 1. Tests for stationarity and chooses d 2. Searches over many combinations of p and q 3. Selects the model with the **lowest AICc** (corrected AIC) 4. Returns the best ARIMA or SARIMA model

It is a good **starting point**, but you should always check residuals and potentially compare manually specified alternatives.
:::

# BLOCK 8 — Tricky Concepts & Common Mistakes

## Things students often get wrong

::: callout-warning
**Common mistake 1:** Confusing seasonal and cyclical components.\
→ If it repeats within a year and is predictable: **seasonal**.\
→ If it takes years and is irregular: **cyclical**.
:::

::: callout-warning
**Common mistake 2:** Using a multiplicative model on data with zero or near-zero values.\
→ Multiplicative models fail when values approach zero (you can't multiply by zero). Use additive or log-transform first.
:::

::: callout-warning
**Common mistake 3:** Thinking a low AIC always means the best model.\
→ AIC and RMSE together tell a fuller story. A slightly higher AIC but much lower RMSE might still be preferred, especially if residuals look better.
:::

::: callout-warning
**Common mistake 4:** Forgetting to check stationarity before ARIMA.\
→ Always plot the series and run `adf.test()` before fitting ARIMA. Never skip this step.
:::

::: callout-warning
**Common mistake 5:** Interpreting the Ljung-Box test backwards.\
→ **p \> 0.05** = residuals ARE random = **good model** ✅\
→ **p \< 0.05** = residuals have structure = **model needs work** ❌
:::
