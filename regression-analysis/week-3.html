Week 3 - Power Analysis + Missing Data

Introduction

This class had two main parts:

  • Power analysis → how likely a statistical test is to detect a real effect
  • Missing data → how to identify, visualise, and deal with missing values in R

These two topics are very important in real data analysis because they answer two practical questions:

  1. Do we have enough data to detect something important?
  2. Can we trust our analysis if some of the data is missing?

A lot of statistics courses focus on how to run a test, but this class goes one step further. It asks whether the test had a fair chance of finding an effect in the first place, and whether missing values may be distorting the result.

Part 1 - Power Analysis

1. What is statistical power?

Statistical power is the probability of rejecting the null hypothesis when the null hypothesis is false.

In symbols:

\[ \text{Power} = 1 - \beta \]

where:

  • ( ) = probability of a Type II error
  • ( 1 - ) = probability of correctly detecting a real effect

How to think about it

Power is basically the test’s ability to notice something real.

Imagine a smoke alarm:

  • A powerful smoke alarm goes off when there really is smoke
  • A weak smoke alarm misses the smoke and stays silent

A statistical test is similar. If there really is an effect in the population, a high-power test is more likely to find it.

2. Why do we need power analysis?

A non-significant result can mean two different things:

  • there really is no effect
  • there is an effect, but the study was too weak to detect it

This is why power matters.

Real-world meaning

Suppose a new medicine really helps patients a little, but your sample is tiny.

Your test may say:

“Fail to reject the null hypothesis”

But that does not necessarily mean the medicine does nothing. It may just mean the study did not have enough power.

That is the main purpose of power analysis:

to check whether the sample size is large enough to detect an effect that matters

3. The four linked quantities

The slides say that four quantities are related:

  • sample size (n)
  • effect size
  • significance level ( )
  • power (1 - )

Why this matters

These four quantities work like a balancing system.

If you change one, the others are affected.

General pattern

  • larger sample size → higher power
  • larger effect size → higher power
  • smaller alpha (stricter test) → lower power
  • higher desired power → usually need larger sample size

Intuition

If you are trying to detect a very small difference, you need more data.

If you are happy to detect only large differences, you may need less data.

So power analysis is really about matching the study design to the kind of effect you care about.

4. Effect size in power analysis

In practice, we do not know the true effect size.

If we already knew the true effect, there would be no need to do the test.

So instead, we choose a minimum effect size of practical importance.

Important idea

This is not just “any difference”.

It is the smallest difference that would actually matter in real life.

Real-world example from the lecture

The slides give an example with premature infants, where a difference of 500 g in weight gain was considered clinically significant.

That means the researchers were saying:

“We are not just interested in any tiny difference. We want to be able to detect a difference of at least 500 g, because that is the minimum difference that matters in practice.”

This is the difference between:

  • statistical significance → is there evidence of some effect?
  • practical or clinical significance → is the effect large enough to matter?

5. Chemical company example

The lecture uses a good example:

A company claims that the mean amount of caustic material is:

\[ \mu = 4.0 \]

An inspector takes a sample of 30 containers and gets:

  • sample mean ( {x} = 4.3 )
  • known standard deviation ( = 1.3 )

If the true amount is greater than 4, the product may be dangerous.

6. Step 1 - State the hypotheses

Because the product is only dangerous if the mean is greater than 4, this is a one-tailed test.

\[ H_0: \mu = 4.0 \]

\[ H_A: \mu > 4.0 \]

Why one-tailed?

Because we only care about too much caustic material.

If the mean were lower than 4, that is not the danger being studied here.

7. Step 2 - Type I and Type II errors in context

Type I error

Reject (H_0) when it is actually true.

In this problem:

We conclude that the true mean is greater than 4 ml per litre when in fact it is not.

So we would falsely label the product as dangerous.

Type II error

Fail to reject (H_0) when it is actually false.

In this problem:

We conclude that the true mean is not greater than 4 ml per litre when in fact it is.

So we would miss a dangerous product.

Why Type II error matters here

This is why power analysis is important.

In safety settings, failing to detect a real danger can be a serious problem.

8. Step 3 - Carry out the hypothesis test

Since the population standard deviation is known, we use a one-sample Z test.

The test statistic is:

\[ Z = \frac{\bar{x} - \mu}{\sigma / \sqrt{n}} \]

Substitute the values:

\[ Z = \frac{4.3 - 4.0}{1.3 / \sqrt{30}} = 1.264 \]

At the 5% significance level for a one-tailed test, the critical value is:

\[ Z_{crit} = 1.645 \]

Since:

\[ 1.264 < 1.645 \]

we fail to reject the null hypothesis.

Conclusion

The data does not provide enough evidence to conclude that the mean level of caustic material is greater than 4 ml per litre.

Important interpretation

This does not prove the mean is 4 or less.

It only means the sample did not give strong enough evidence to conclude it is greater than 4.

That distinction is very important.

9. p-value and lower confidence bound

The lecture also gives:

  • ( p = 0.103 )
  • 95% lower confidence bound = 3.910

Since ( p > 0.05 ), the result is not statistically significant.

The lower confidence bound means:

We are 95% confident that the true population mean is greater than 3.910.

Why this is useful

The p-value only helps with the decision.

The confidence bound gives more information about the range of plausible values.

10. The key power question

After failing to reject the null, the lecturer asks the really important question:

Did the result show there is no dangerous increase, or was the sample just too small to detect it?

That is exactly what power analysis helps answer.

11. Calculating power for a meaningful effect

Suppose we decide that a dangerous increase would be:

\[ 4.35 \text{ ml per litre} \]

That means the effect size of interest is:

\[ 0.35 \]

Now the question becomes:

If the true mean really were 4.35, how likely would this test be to detect it?

That probability is the power.

12. Rejection rule in terms of the sample mean

Earlier we had the critical Z value:

\[ Z_{crit} = 1.645 \]

We can convert this back to the sample-mean scale:

\[ \bar{X} = \mu + \frac{\sigma}{\sqrt{n}} Z_{crit} \]

Substituting:

\[ \bar{X} = 4 + \frac{1.3}{\sqrt{30}}(1.645) = 4.39 \]

Interpretation

  • If ( {X} > 4.39 ) → reject (H_0)
  • If ( {X} < 4.39 ) → fail to reject (H_0)

So the sample mean must be at least 4.39 for the test to detect the effect.

Intuition

This is the “cut-off point” needed before the evidence is strong enough.

13. Calculating Type II error and power

If the true mean is actually (4.35), then Type II error means:

we fail to reject the null even though the alternative is true

So we need:

\[ P(\bar{X} < 4.39 \mid \mu = 4.35) \]

Convert to a Z score:

\[ Z = \frac{4.39 - 4.35}{1.3/\sqrt{30}} = 0.17 \]

From the normal table:

\[ P(Z < 0.17) = 0.5675 \]

So:

\[ \beta = 0.5675 \]

and therefore:

\[ \text{Power} = 1 - \beta = 1 - 0.5675 = 0.4325 \]

So the power is:

\[ 43.25\% \]

14. What does power = 43.25% mean?

This means:

If the true mean is really 4.35, this test would detect that effect only about 43% of the time.

That is not very good.

Interpretation

There is a bigger chance of missing the effect than detecting it.

So this low power suggests the study may be too weak to reliably detect that effect.

Real-world meaning

If you are testing something potentially dangerous, 43% power is poor.

You would want a better chance of catching the problem.

Usually researchers want power around 80% or higher, though the lecture did not focus on that threshold explicitly.

15. Why smaller effects are harder to detect

The exercise then changes the effect size from 0.35 to 0.1.

The answer becomes much lower power.

Why?

Because smaller effects are harder to separate from random noise.

Real-world analogy

It is easier to hear a loud alarm in a noisy room than a very quiet beep.

A large effect stands out more clearly. A small effect gets lost more easily.

Part 2 - Power, confidence intervals, and margin of error

16. Confidence interval formula

When the population standard deviation is known, the confidence interval for a mean is:

\[ \bar{x} \pm Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}} \]

The term

\[ Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}} \]

is the margin of error (ME).

17. What is margin of error?

The margin of error is how far we are willing for our estimate to be from the true value.

Intuition

If the margin of error is small, we want a very precise estimate.

If we want a more precise estimate, we usually need a larger sample size.

So margin of error and sample size are directly connected.

18. Sample size formula from margin of error

Starting with:

\[ ME = Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}} \]

rearranging gives:

\[ n = \left(\frac{Z_{\alpha/2}\sigma}{ME}\right)^2 \]

What this means

If you know:

  • your confidence level
  • your standard deviation
  • your desired margin of error

you can work out how much data you need.

19. Sample size example

Suppose an analyst wants to estimate mean voltage.

Given:

  • population standard deviation ( = 22.5 )
  • margin of error ( ME = 5 )
  • 95% confidence level, so ( Z = 1.96 )

Then:

\[ n = \left(\frac{1.96 \times 22.5}{5}\right)^2 \]

This gives:

\[ n = 77.8 \]

Since sample size must be a whole number, we round up:

\[ n = 78 \]

Why round up?

Because 77 would not quite give the precision we need.

When planning sample size, always round up.

20. Unknown standard deviation

In real life, the population standard deviation is often unknown.

Then we use the sample standard deviation (s) instead, and the distribution is based on t rather than Z.

That is more realistic, but also a little more complicated.

The main idea is still the same:

more precision requires more data

Part 3 - Missing Data

21. Why missing data matters

Missing data is not just annoying. It can change your results.

If missing values are ignored carelessly, you may:

  • lose a lot of data
  • reduce sample size
  • reduce power
  • introduce bias
  • fit models on a non-representative subset

So missing data is really about data quality, not just cleanup.

22. Useful R functions for small amounts of missing data

The slides list several useful functions when there are only a few missing values and no obvious missing-data pattern.

is.na(x)

Checks whether each value is missing.

It returns:

  • TRUE if missing
  • FALSE if not missing
### `is.na(x)`

Checks whether each value is missing.

It returns:

  • TRUE if missing
  • FALSE if not missing
is.na(x)

🧠 How to think about it

This is the basic building block for missing data.

👉 You are asking:

“Where exactly is my data missing?”

sum(is.na(x))

Counts the number of missing values.

sum(is.na(x))

🧠 Why this is useful

Instead of looking at TRUE/FALSE values, this gives you a quick summary.

👉 Example:

  • If result = 5 → there are 5 missing values

mean(is.na(x))

Gives the proportion of missing values.

mean(is.na(x))

🧠 Why this matters

  • If result = 0.10 → 10% of data is missing
  • This helps you judge how serious the problem is

complete.cases(x)

Returns rows with no missing values.

complete.cases(x)

Removing missing data

x_clean <- x[complete.cases(x), ]

🧠 Important warning

This method is called listwise deletion.

It is simple, but:

  • reduces sample size
  • reduces power
  • may introduce bias

👉 If missing data is not random, this can be very dangerous.

23. Types of missing data (VERY IMPORTANT)

Not all missing data is the same.

This is critical for your understanding.

1. MCAR — Missing Completely At Random

Missingness has no relationship with any variable.

👉 Example:

  • A survey page is accidentally lost

✔ Safe to remove data (usually)

2. MAR — Missing At Random

Missingness depends on observed variables

👉 Example:

  • Income missing more often for younger people

✔ Can be handled with proper methods (e.g. imputation)

3. MNAR — Missing Not At Random

Missingness depends on the missing value itself

👉 Example:

  • People with very high income choose not to report it

❗ Most problematic case

🧠 Why this matters

👉 Because:

The method you choose depends on why the data is missing

24. Visualising missing data

The lecture uses the VIM package.

This is VERY useful in practice.

aggr() function

library(VIM)
aggr(data)

What it shows

  • Amount of missing data per variable
  • Patterns of missingness

🧠 How to interpret

  • Bars → how much is missing

  • Colours:

    • Blue = observed
    • Red = missing

👉 Helps answer:

“Which variables have the most missing data?”

barMiss()

barMiss(data)

What it does

Shows missing values grouped by categories

👉 Example:

  • Missing income by gender

🧠 Why this is powerful

Helps detect patterns

👉 If missing data is not random → this will show it

Scatter plots with missing data

  • Red points = missing values
  • Boxplots show distribution

👉 Helps understand:

Is missingness related to other variables?

25. What to do with missing data?

There is no single answer.

But here are common approaches:

1. Remove data (simple approach)

✔ Easy ❌ Can lose a lot of information

2. Imputation (fill in missing values)

Examples:

  • Mean imputation
  • Median imputation

👉 Replace missing values with a reasonable estimate

⚠️ Important limitation

  • Reduces variability
  • Can distort relationships

3. More advanced methods (not covered deeply yet)

  • Multiple imputation
  • Model-based methods

👉 You will likely see these later in the course

26. Key connection: Missing data and power

This is VERY important and often overlooked.

👉 Missing data → smaller effective sample size

Which means:

  • higher standard error
  • wider confidence intervals
  • lower power

🧠 Big picture insight

Missing data is not just a “data cleaning issue”.

It directly affects:

  • inference
  • hypothesis testing
  • reliability of conclusions

🔚 Final Takeaways (Week 3)

Power Analysis

  • Power = probability of detecting a real effect

  • Depends on:

    • sample size
    • effect size
    • significance level

👉 Low power = high chance of missing real effects

Confidence and Sample Size

  • Margin of error controls precision
  • Larger sample → smaller error
  • Can calculate required sample size before study

Missing Data

  • Always check it first
  • Understand why it is missing
  • Be careful when removing data

🧠 Most important idea from this lecture

A non-significant result does NOT mean “no effect”

It could mean:

  • no effect
  • OR not enough power
  • OR poor data (missing values)
