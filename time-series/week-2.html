2 Core Explanation

2.1 Time Series Components — Recap

A time series is a sequence of observations collected at equally spaced time points. The key idea is that we try to understand the pattern in the data before making forecasts.

A time series \(y_t\) can be decomposed into four main components:

  • Trend (\(T_t\)): The long-term increase or decrease in the data. Can be linear, exponential, or more complex.
  • Seasonal variation (\(S_t\)): A periodic pattern that repeats within a fixed calendar period (e.g., every year). The variation completes itself within one year.
  • Cyclical variation: A non-periodic, longer-term (> 2 years) wave-like pattern. Unlike seasonality, cycles don’t have a fixed length. In practice, these are often absorbed into the trend component in our models.
  • Residual/Random variation (\(R_t\)): Everything that cannot be explained by the three components above. This is the stochastic (random) part. If this component still shows a pattern, something has gone wrong in the decomposition.

Important from the lecture (Aula_2): The first three components are deterministic — they follow mathematical patterns we can model. The residual is stochastic — it should look like white noise (random, centred around zero, normally distributed). If your residuals show a pattern, you have not correctly identified trend or seasonality.

2.1.1 The Periodicity Concept

Periodicity (\(p\)) = the number of observations in one complete seasonal cycle.

Data frequency Periodicity
Monthly 12
Quarterly 4
Weekly (Mon–Fri) 5
Weekly (Mon–Sun) 7
Hourly (daily pattern) 24
Annual No seasonality

Rule of thumb from the lecture: You need at least 3–4 complete seasonal cycles worth of data to reliably estimate trend and seasonality. For monthly data that means ≥ 36 observations.

2.2 Time Series Decomposition

2.2.1 The Decomposition Model

\[y_t = f(S_t,\ T_t,\ R_t)\]

The choice of how to combine the components depends on whether the seasonal variation is constant or growing:

Model Formula When to use
Additive \(y_t = S_t + T_t + R_t\) Seasonal amplitude is constant (homogeneous seasonality)
Multiplicative \(y_t = S_t \times T_t \times R_t\) Seasonal amplitude grows with the level (heterogeneous / heteroscedastic)

Tip: If your time series shows increasing “swings” as the level goes up (like sales growing with bigger peaks every year), use multiplicative. If the swings stay roughly the same size, use additive.

2.2.2 Converting Multiplicative to Additive (Box-Cox / log transform)

A multiplicative model can be converted to additive by taking logs:

\[\log(y_t) = \log(S_t) + \log(T_t) + \log(R_t)\]

This is a special case of the Box-Cox transformation, which can be applied whenever you need to stabilise the variance.

2.3 Seasonal Adjustment

Once we have an estimate of the seasonal component, we can remove it to study the underlying trend more clearly:

Model Seasonally adjusted series
Additive \(y_t - \hat{S}_t = T_t + R_t\)
Multiplicative \(y_t / \hat{S}_t = T_t \times R_t\)

Warning from the lecture slides: A seasonally adjusted series still contains the trend and the residuals, so it is not smooth — it can show apparent “ups and downs” that are actually just noise. Use the trend-cycle component if you want to see inflection points more clearly.

2.3.1 Key R functions for decomposition

library(forecast)

# Classical decomposition (additive or multiplicative)
births.ts.comp <- decompose(births.ts)
plot(births.ts.comp)

# Extract individual components
births.ts.comp$seasonal   # seasonal component
births.ts.comp$trend      # trend component
births.ts.comp$random     # random/residual component

# Seasonally adjust
births.ts.sadj <- births.ts - births.ts.comp$seasonal  # additive
plot(births.ts.sadj)

# STL components (see Section 3.3)
seasonal(fit)     # seasonal component
trendcycle(fit)   # trend-cycle component
remainder(fit)    # residual component
seasadj(fit)      # seasonally adjusted series

2.4 STL Decomposition

STL stands for “Seasonal and Trend decomposition using Loess”. It is the recommended method for most practical work.

2.4.1 Advantages of STL over classical decomposition

  • Robust to outliers (extreme values have less influence)
  • Can handle any type of seasonality
  • The user can control how “wiggly” the trend and seasonal components are via the window parameters

2.4.2 Disadvantages

  • Designed for additive models only (apply log/Box-Cox for multiplicative)
  • Does not handle calendar effects automatically

2.4.3 Key parameters

Parameter Controls
s.window How much the seasonal component is allowed to change over time. Use "periodic" to fix it constant.
t.window How smooth the trend component is.
robust = TRUE Makes the decomposition resistant to outliers.

2.4.4 R code

library(fpp2)

# STL with fixed seasonal component
fit <- stl(elecequip, s.window = "periodic")
autoplot(fit) + xlab("year")

# STL with slowly changing seasonality and robust option
fit <- stl(elecequip, s.window = 7, robust = TRUE)
autoplot(fit) + xlab("year")

# Adjust t.window and s.window manually
fit <- stl(elecequip, t.window = 15, s.window = "periodic", robust = TRUE)

# mstl(): automatically selects s.window; supports Box-Cox via lambda
elecequip %>% mstl() %>% autoplot()
elecequip %>% mstl(lambda = "auto") %>% autoplot()  # with Box-Cox

2.5 Prediction Using Decomposition

The typical workflow is:

  1. Decompose the series with STL to understand its structure.
  2. Seasonally adjust the series (remove \(\hat{S}_t\)).
  3. Apply a non-seasonal forecast method (e.g., naive, SES, Holt) to the adjusted series.
  4. Re-add the seasonal component to produce the final forecast.

The seasonal component is forecast by repeating the last observed seasonal pattern (naive seasonal forecast).

2.5.1 R code example

fit <- stl(elecequip, t.window = 13, s.window = "periodic")

# Forecast using naive method on seasonally adjusted data
fit %>% seasadj() %>% naive() %>%
  autoplot() + ylab("New orders index") +
  ggtitle("Seasonal adjusted data ETS Predictions")

# Shorthand: stlf() does all steps automatically
elecequip %>% stlf(method = 'naive') %>%
  autoplot() + ylab("New orders index") + xlab("Year")

2.6 Simple Moving Average (SMA)

The Simple Moving Average smooths a time series by replacing each value with the average of the surrounding \(p\) values. It is the foundation of trend estimation in classical decomposition.

2.6.1 Formula

\[F_t = MA_p = \frac{\sum_{i=1}^{p} D_i}{p}\]

where \(F_t\) is the prediction, \(D_i\) are the observations, and \(p\) is the periodicity.

2.6.2 How to calculate it manually (from the Excel lecture)

  1. Identify the periodicity \(p\).
  2. Take a block of \(p\) consecutive observations.
  3. Calculate the mean of that block.
  4. Place the mean at the centre of the block.
  5. Slide the window one step forward and repeat.

Important: You lose \(\lfloor p/2 \rfloor\) values at the start and end of the series because there is no full block available there.

Special case — even \(p\): When \(p\) is even (e.g., \(p=4\) for quarterly data), there is no integer centre. You need two steps: calculate \(MA_p\), then calculate \(MA_2\) of consecutive \(MA_p\) values. This centres the estimate correctly.

2.6.3 R code

library(forecast)
library(TTR)   # also provides SMA

sm <- ma(wineind, order = 12)
plot(wineind)
lines(sm, col = "red")

2.7 Exponential Smoothing — Introduction

Exponential smoothing is a family of pure time series methods — they use only the history of the variable being forecast, with no external predictors.

Key idea: Recent observations should receive more weight than older ones. The weights decrease exponentially as observations get older.

2.7.1 When to use exponential smoothing

  • When you only have data for the series itself (no independent variables)
  • When there are too few variables to build a causal model
  • When quick forecasts are needed with reasonable accuracy

2.7.2 Types of Exponential Smoothing

Method Trend Seasonality
SES (Simple Exponential Smoothing) None None
Holt Linear Additive (linear) None
Holt Damped Damped (flattens out) None
Holt-Winters Additive Additive Additive
Holt-Winters Multiplicative Additive Multiplicative
Holt-Winters Damped Damped Multiplicative

2.8 Simple Exponential Smoothing (SES)

Use when: Series has no trend and no seasonality.

2.8.1 The Two Key Equations

Forecast equation: \[\hat{y}_{t+h|t} = \ell_t\]

Smoothing (level) equation: \[\ell_t = \alpha y_t + (1 - \alpha)\ell_{t-1}, \quad 0 \leq \alpha \leq 1\]

\(\ell_t\) is called the level (or exponential smoothing) of the series.

By expanding the smoothing equation recursively: \[\hat{y}_{T+1|T} = \sum_{j=0}^{T-1} \alpha(1-\alpha)^j y_{T-j} + (1-\alpha)^T \ell_0\]

This shows the forecast is a weighted average of all past observations, with weights declining exponentially.

2.8.2 The smoothing parameter \(\alpha\)

\(\alpha\) value Effect
Close to 1 Nearly all weight on the most recent observation (reactive, under-smoothed)
Close to 0 Weight spread equally across all past observations (smooth, slow to adapt)

2.8.3 Choosing \(\alpha\) and \(\ell_0\) — Optimisation

Optimal values are found by minimising the Sum of Squared Errors (SSE):

\[\text{SSE} = \sum_{t=1}^{T}(y_t - \hat{y}_{t|t-1})^2 = \sum_{t=1}^{T} e_t^2\]

This is a non-linear minimisation — R handles it numerically.

2.8.4 ETS notation: ETS(A,N,N)

SES is the ETS model with Additive error, No trend, No seasonality.

2.8.5 R code

library(forecast)

# Fit SES — R optimises alpha automatically
fc <- ses(oildata, h = 5)
summary(fc)

# Plot forecast with fitted values
autoplot(fc) +
  autolayer(fitted(fc), series = "Fitted") +
  ylab("oil (1M Tones)") + xlab("year")

# Using ets() for more control
SES <- ets(train.ts, model = "ANN", alpha = 0.2)
# ETS(A,N,N): error = Additive, Trend = None, Seasonality = None

2.9 Trend Methods

2.9.1 Holt’s Linear Trend Method

Use when: Series has a linear trend but no seasonality.

Extends SES with a slope (trend) equation:

Forecast equation: \[\hat{y}_{t+h|t} = \ell_t + h b_t\]

Level equation: \[\ell_t = \alpha y_t + (1-\alpha)(\ell_{t-1} + b_{t-1})\]

Trend (slope) equation: \[b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)b_{t-1}\]

Parameters: \(\alpha, \beta^* \in [0,1]\). Choose \(\alpha, \beta^*, \ell_0, b_0\) to minimise SSE.

2.9.2 Holt’s Damped Trend Method

Problem with Holt Linear: Forecasts trend indefinitely into the future — often too optimistic long term.

Solution: Introduce a damping parameter \(\phi\) that flattens the trend over time:

\[\hat{y}_{t+h|t} = \ell_t + (\phi + \phi^2 + \cdots + \phi^h)b_t\]

\[\ell_t = \alpha y_t + (1-\alpha)(\ell_{t-1} + \phi b_{t-1})\]

\[b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)\phi b_{t-1}\]

  • \(0 < \phi < 1\): dampened trend (recommended for long-horizon forecasts)
  • \(\phi = 1\): equivalent to Holt’s linear trend

2.9.3 R code

library(fpp2)

air_data <- window(ausair, start = 1990, end = 2004)

# Holt's linear trend
air_data %>% holt(h = 5, PI = FALSE) %>% autoplot()

# Holt's damped trend
air_data %>% holt(damped = TRUE, h = 5, PI = FALSE) %>% autoplot()

# Compare models on livestock data
livestock2 <- window(livestock, start = 1970, end = 2000)
fit1 <- ses(livestock2)
fit2 <- holt(livestock2)
fit3 <- holt(livestock2, damped = TRUE)

# Compare accuracy
accuracy(fit1, livestock)
accuracy(fit2, livestock)
accuracy(fit3, livestock)

2.10 Seasonal Methods — Holt-Winters

Extends Holt to include a seasonal component \(s_t\).

2.10.1 Holt-Winters Additive

Use when: Seasonal amplitude is constant (does not grow with the level).

\[\hat{y}_{t+h|t} = \ell_t + h b_t + s_{t+h-m(k+1)}\]

\[\ell_t = \alpha(y_t - s_{t-m}) + (1-\alpha)(\ell_{t-1} + b_{t-1})\]

\[b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)b_{t-1}\]

\[s_t = \gamma(y_t - \ell_{t-1} - b_{t-1}) + (1-\gamma)s_{t-m}\]

Parameters: \(0 \leq \alpha \leq 1\), \(0 \leq \beta^* \leq 1\), \(0 \leq \gamma \leq 1-\alpha\)

\(m\) = periodicity (e.g., 12 for monthly), \(k = \lfloor(h-1)/m\rfloor\)

2.10.2 Holt-Winters Multiplicative

Use when: Seasonal amplitude grows proportionally with the level.

\[\hat{y}_{t+h|t} = (\ell_t + h b_t) s_{t+h-m(k+1)}\]

\[\ell_t = \alpha \frac{y_t}{s_{t-m}} + (1-\alpha)(\ell_{t-1} + b_{t-1})\]

\[s_t = \gamma \frac{y_t}{\ell_{t-1} + b_{t-1}} + (1-\gamma)s_{t-m}\]

2.10.3 Holt-Winters Damped

Usually the best performing method for seasonal series. Adds the damping parameter \(\phi\) to the multiplicative seasonal model.

2.10.4 R code

library(fpp2)

aust <- window(austourists, start = 2005)

# Fit additive and multiplicative Holt-Winters
fit1 <- hw(aust, seasonal = "additive")
fit2 <- hw(aust, seasonal = "multiplicative")

# Plot both
tmp <- cbind(Data = aust,
             "HW additive"       = fit1[["mean"]],
             "HW multiplicative" = fit2[["mean"]])
autoplot(tmp) + xlab("Year") + ylab("Hotel Stays (millions)")

# Damped multiplicative (usually best for seasonal TS)
fc <- hw(subset(hyndsight, end = length(hyndsight) - 35),
         damped = TRUE, seasonal = "multiplicative", h = 35)

# Extract estimated components
addstates  <- fit1$model$states[, 1:3]
multstates <- fit2$model$states[, 1:3]
colnames(addstates) <- colnames(multstates) <- c("level","slope","seasonality")

2.11 Prediction Error Measurements

These metrics allow you to compare competing models on a test set.

Metric Formula Notes
MAE / MAD \(\frac{\sum |Y_t - F_t|}{n}\) Same units as data; easy to interpret
MSE \(\frac{\sum (Y_t - F_t)^2}{n-1}\) Penalises large errors more heavily
MAPE \(\frac{1}{n}\sum\left|\frac{Y_t - F_t}{Y_t}\right| \times 100\) Percentage; unit-free; allows comparison across series
RMSE \(\sqrt{MSE}\) Same units as data; commonly reported
MASE Scaled MAE Useful when comparing across different scales

Also used: AIC, AICc, BIC — information criteria for model selection (lower = better).

From the livestock example: The Holt Linear method outperformed both SES and Holt Damped on the test set (lower RMSE, MAE, MAPE). Always evaluate on a held-out test set, not just training data.

2.12 ETS Framework in R

ETS = Error, Trend, Seasonality. This unified framework encompasses all exponential smoothing methods.

Each component takes a type:

Component Types
Error A (additive), M (multiplicative)
Trend N (none), A (additive), Ad (additive damped)
Seasonality N (none), A (additive), M (multiplicative)

Examples: ETS(A,N,N) = SES; ETS(A,A,A) = Holt-Winters additive; ETS(M,Ad,M) = Holt-Winters damped multiplicative.

2.12.1 R code

library(fpp2)

aust <- window(austourists, start = 2005)

# Auto-select model by AICc
fit <- ets(aust)
summary(fit)
autoplot(fit)

# Force a specific model
ets(h02, model = "AAA", damped = FALSE)

# Train/test split workflow
train <- window(h02, end  = c(2004, 12))
test  <- window(h02, start = 2005)

fit1 <- ets(train)                                    # fit on train
fit2 <- ets(test, model = fit1, use.initial.values = TRUE)  # apply to test

accuracy(fit2)
accuracy(forecast(fit1, 10), test)

# Residual diagnostics
cbind('Residuals'      = residuals(fit),
      'Forecast errors' = residuals(fit, type = 'response')) %>%
  autoplot(facet = TRUE) + xlab("Year") + ylab("")

3 Key Equations/Formulas Summary

Decomposition: \[y_t = S_t + T_t + R_t \quad \text{(additive)} \qquad y_t = S_t \times T_t \times R_t \quad \text{(multiplicative)}\]

Simple Moving Average: \[MA_p = \frac{\sum_{i=1}^{p} D_i}{p}\]

SES forecast & smoothing: \[\hat{y}_{t+h|t} = \ell_t \qquad \ell_t = \alpha y_t + (1-\alpha)\ell_{t-1}\]

Holt’s linear trend: \[\hat{y}_{t+h|t} = \ell_t + hb_t \qquad b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)b_{t-1}\]

Optimisation criterion (SSE): \[\text{SSE} = \sum_{t=1}^{T}(y_t - \hat{y}_{t|t-1})^2\]

Error metrics: \[\text{MAE} = \frac{\sum|Y_t - F_t|}{n} \qquad \text{MAPE} = \frac{1}{n}\sum\left|\frac{Y_t-F_t}{Y_t}\right|\times100\]

5 Summary Points

  • A time series is decomposed into Trend + Seasonal + Cyclical + Residual. The cyclical component is usually absorbed into the trend in practice.
  • Choose additive decomposition for constant seasonality; multiplicative (or log-transform) for growing seasonality.
  • STL is preferred over classical decomposition — it is robust to outliers and gives user control over smoothness.
  • The residuals after decomposition must look random (no pattern, centred on zero, approximately normal). If they don’t, the decomposition is incomplete.
  • SES is for series with no trend and no seasonality. The parameter \(\alpha\) controls how quickly the model “forgets” old data.
  • Holt’s method adds a trend component (linear or damped). The damped version is safer for long-horizon forecasts.
  • Holt-Winters adds a seasonal component on top of Holt. The damped multiplicative version is often the best performer.
  • Always evaluate models on a test set using error metrics (MAE, RMSE, MAPE). Minimise AICc for model selection within the ETS framework.
  • ets() in R selects the best ETS model automatically using AICc — but use ses(), holt(), and hw() when you want direct control.

# Core Explanation

## Time Series Components — Recap

A **time series** is a sequence of observations collected at equally spaced time points. The key idea is that we try to **understand the pattern** in the data before making forecasts.

A time series $y_t$ can be decomposed into four main components:

- **Trend ($T_t$):** The long-term increase or decrease in the data. Can be linear, exponential, or more complex.
- **Seasonal variation ($S_t$):** A **periodic** pattern that repeats within a fixed calendar period (e.g., every year). *The variation completes itself within one year.*
- **Cyclical variation:** A **non-periodic**, longer-term (> 2 years) wave-like pattern. Unlike seasonality, cycles don't have a fixed length. In practice, these are often absorbed into the trend component in our models.
- **Residual/Random variation ($R_t$):** Everything that cannot be explained by the three components above. This is the **stochastic (random) part**. If this component still shows a pattern, something has gone wrong in the decomposition.

> **Important from the lecture (Aula_2):** The first three components are *deterministic* — they follow mathematical patterns we can model. The residual is *stochastic* — it should look like white noise (random, centred around zero, normally distributed). If your residuals show a pattern, you have not correctly identified trend or seasonality.

### The Periodicity Concept

**Periodicity ($p$)** = the number of observations in one complete seasonal cycle.

| Data frequency | Periodicity |
|---|---|
| Monthly | 12 |
| Quarterly | 4 |
| Weekly (Mon–Fri) | 5 |
| Weekly (Mon–Sun) | 7 |
| Hourly (daily pattern) | 24 |
| Annual | No seasonality |

> **Rule of thumb from the lecture:** You need **at least 3–4 complete seasonal cycles** worth of data to reliably estimate trend and seasonality. For monthly data that means ≥ 36 observations.

---

## Time Series Decomposition

### The Decomposition Model

$$y_t = f(S_t,\ T_t,\ R_t)$$

The choice of how to *combine* the components depends on whether the **seasonal variation is constant or growing**:

| Model | Formula | When to use |
|---|---|---|
| **Additive** | $y_t = S_t + T_t + R_t$ | Seasonal amplitude is **constant** (homogeneous seasonality) |
| **Multiplicative** | $y_t = S_t \times T_t \times R_t$ | Seasonal amplitude **grows** with the level (heterogeneous / heteroscedastic) |

> **Tip:** If your time series shows increasing "swings" as the level goes up (like sales growing with bigger peaks every year), use multiplicative. If the swings stay roughly the same size, use additive.

### Converting Multiplicative to Additive (Box-Cox / log transform)

A multiplicative model can be converted to additive by taking logs:

$$\log(y_t) = \log(S_t) + \log(T_t) + \log(R_t)$$

This is a special case of the **Box-Cox transformation**, which can be applied whenever you need to stabilise the variance.

---

## Seasonal Adjustment

Once we have an estimate of the seasonal component, we can **remove it** to study the underlying trend more clearly:

| Model | Seasonally adjusted series |
|---|---|
| Additive | $y_t - \hat{S}_t = T_t + R_t$ |
| Multiplicative | $y_t / \hat{S}_t = T_t \times R_t$ |

> **Warning from the lecture slides:** A seasonally adjusted series still contains the trend *and* the residuals, so it is **not smooth** — it can show apparent "ups and downs" that are actually just noise. Use the **trend-cycle component** if you want to see inflection points more clearly.

### Key R functions for decomposition

```r
library(forecast)

# Classical decomposition (additive or multiplicative)
births.ts.comp <- decompose(births.ts)
plot(births.ts.comp)

# Extract individual components
births.ts.comp$seasonal   # seasonal component
births.ts.comp$trend      # trend component
births.ts.comp$random     # random/residual component

# Seasonally adjust
births.ts.sadj <- births.ts - births.ts.comp$seasonal  # additive
plot(births.ts.sadj)

# STL components (see Section 3.3)
seasonal(fit)     # seasonal component
trendcycle(fit)   # trend-cycle component
remainder(fit)    # residual component
seasadj(fit)      # seasonally adjusted series
```

## STL Decomposition

**STL** stands for *"Seasonal and Trend decomposition using Loess"*. It is the recommended method for most practical work.

### Advantages of STL over classical decomposition

- **Robust** to outliers (extreme values have less influence)
- Can handle any type of seasonality
- The user can control how "wiggly" the trend and seasonal components are via the window parameters

### Disadvantages

- Designed for **additive models only** (apply log/Box-Cox for multiplicative)
- Does not handle calendar effects automatically

### Key parameters

| Parameter | Controls |
|---|---|
| `s.window` | How much the seasonal component is allowed to change over time. Use `"periodic"` to fix it constant. |
| `t.window` | How smooth the trend component is. |
| `robust = TRUE` | Makes the decomposition resistant to outliers. |

### R code

```r
library(fpp2)

# STL with fixed seasonal component
fit <- stl(elecequip, s.window = "periodic")
autoplot(fit) + xlab("year")

# STL with slowly changing seasonality and robust option
fit <- stl(elecequip, s.window = 7, robust = TRUE)
autoplot(fit) + xlab("year")

# Adjust t.window and s.window manually
fit <- stl(elecequip, t.window = 15, s.window = "periodic", robust = TRUE)

# mstl(): automatically selects s.window; supports Box-Cox via lambda
elecequip %>% mstl() %>% autoplot()
elecequip %>% mstl(lambda = "auto") %>% autoplot()  # with Box-Cox
```

## Prediction Using Decomposition

The typical workflow is:

1. **Decompose** the series with STL to understand its structure.
2. **Seasonally adjust** the series (remove $\hat{S}_t$).
3. **Apply a non-seasonal forecast method** (e.g., naive, SES, Holt) to the adjusted series.
4. **Re-add** the seasonal component to produce the final forecast.

> The seasonal component is forecast by repeating the **last observed seasonal pattern** (naive seasonal forecast).

### R code example

```r
fit <- stl(elecequip, t.window = 13, s.window = "periodic")

# Forecast using naive method on seasonally adjusted data
fit %>% seasadj() %>% naive() %>%
  autoplot() + ylab("New orders index") +
  ggtitle("Seasonal adjusted data ETS Predictions")

# Shorthand: stlf() does all steps automatically
elecequip %>% stlf(method = 'naive') %>%
  autoplot() + ylab("New orders index") + xlab("Year")
```

## Simple Moving Average (SMA)

The **Simple Moving Average** smooths a time series by replacing each value with the average of the surrounding $p$ values. It is the foundation of trend estimation in classical decomposition.

### Formula

$$F_t = MA_p = \frac{\sum_{i=1}^{p} D_i}{p}$$

where $F_t$ is the prediction, $D_i$ are the observations, and $p$ is the periodicity.

### How to calculate it manually (from the Excel lecture)

1. Identify the **periodicity** $p$.
2. Take a **block** of $p$ consecutive observations.
3. Calculate the **mean** of that block.
4. Place the mean at the **centre** of the block.
5. Slide the window one step forward and repeat.

> **Important:** You lose $\lfloor p/2 \rfloor$ values at the **start** and **end** of the series because there is no full block available there.

**Special case — even $p$:** When $p$ is even (e.g., $p=4$ for quarterly data), there is no integer centre. You need **two steps**: calculate $MA_p$, then calculate $MA_2$ of consecutive $MA_p$ values. This centres the estimate correctly.

### R code

```r
library(forecast)
library(TTR)   # also provides SMA

sm <- ma(wineind, order = 12)
plot(wineind)
lines(sm, col = "red")
```

## Exponential Smoothing — Introduction

Exponential smoothing is a family of **pure time series methods** — they use only the history of the variable being forecast, with no external predictors.

**Key idea:** Recent observations should receive **more weight** than older ones. The weights decrease *exponentially* as observations get older.

### When to use exponential smoothing

- When you only have data for the series itself (no independent variables)
- When there are too few variables to build a causal model
- When quick forecasts are needed with reasonable accuracy

### Types of Exponential Smoothing

| Method | Trend | Seasonality |
|---|---|---|
| **SES** (Simple Exponential Smoothing) | None | None |
| **Holt Linear** | Additive (linear) | None |
| **Holt Damped** | Damped (flattens out) | None |
| **Holt-Winters Additive** | Additive | Additive |
| **Holt-Winters Multiplicative** | Additive | Multiplicative |
| **Holt-Winters Damped** | Damped | Multiplicative |

---

## Simple Exponential Smoothing (SES)

**Use when:** Series has **no trend** and **no seasonality**.

### The Two Key Equations

**Forecast equation:**
$$\hat{y}_{t+h|t} = \ell_t$$

**Smoothing (level) equation:**
$$\ell_t = \alpha y_t + (1 - \alpha)\ell_{t-1}, \quad 0 \leq \alpha \leq 1$$

$\ell_t$ is called the **level** (or exponential smoothing) of the series.

By expanding the smoothing equation recursively:
$$\hat{y}_{T+1|T} = \sum_{j=0}^{T-1} \alpha(1-\alpha)^j y_{T-j} + (1-\alpha)^T \ell_0$$

This shows the forecast is a **weighted average of all past observations**, with weights declining exponentially.

### The smoothing parameter $\alpha$

| $\alpha$ value | Effect |
|---|---|
| Close to **1** | Nearly all weight on the most recent observation (reactive, under-smoothed) |
| Close to **0** | Weight spread equally across all past observations (smooth, slow to adapt) |

### Choosing $\alpha$ and $\ell_0$ — Optimisation

Optimal values are found by **minimising the Sum of Squared Errors (SSE)**:

$$\text{SSE} = \sum_{t=1}^{T}(y_t - \hat{y}_{t|t-1})^2 = \sum_{t=1}^{T} e_t^2$$

This is a non-linear minimisation — R handles it numerically.

### ETS notation: ETS(A,N,N)

SES is the ETS model with **A**dditive error, **N**o trend, **N**o seasonality.

### R code

```r
library(forecast)

# Fit SES — R optimises alpha automatically
fc <- ses(oildata, h = 5)
summary(fc)

# Plot forecast with fitted values
autoplot(fc) +
  autolayer(fitted(fc), series = "Fitted") +
  ylab("oil (1M Tones)") + xlab("year")

# Using ets() for more control
SES <- ets(train.ts, model = "ANN", alpha = 0.2)
# ETS(A,N,N): error = Additive, Trend = None, Seasonality = None
```

## Trend Methods

### Holt's Linear Trend Method

**Use when:** Series has a **linear trend** but **no seasonality**.

Extends SES with a slope (trend) equation:

**Forecast equation:**
$$\hat{y}_{t+h|t} = \ell_t + h b_t$$

**Level equation:**
$$\ell_t = \alpha y_t + (1-\alpha)(\ell_{t-1} + b_{t-1})$$

**Trend (slope) equation:**
$$b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)b_{t-1}$$

Parameters: $\alpha, \beta^* \in [0,1]$. Choose $\alpha, \beta^*, \ell_0, b_0$ to minimise SSE.

### Holt's Damped Trend Method

**Problem with Holt Linear:** Forecasts trend indefinitely into the future — often too optimistic long term.

**Solution:** Introduce a **damping parameter $\phi$** that flattens the trend over time:

$$\hat{y}_{t+h|t} = \ell_t + (\phi + \phi^2 + \cdots + \phi^h)b_t$$

$$\ell_t = \alpha y_t + (1-\alpha)(\ell_{t-1} + \phi b_{t-1})$$

$$b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)\phi b_{t-1}$$

- $0 < \phi < 1$: dampened trend (recommended for long-horizon forecasts)
- $\phi = 1$: equivalent to Holt's linear trend

### R code

```r
library(fpp2)

air_data <- window(ausair, start = 1990, end = 2004)

# Holt's linear trend
air_data %>% holt(h = 5, PI = FALSE) %>% autoplot()

# Holt's damped trend
air_data %>% holt(damped = TRUE, h = 5, PI = FALSE) %>% autoplot()

# Compare models on livestock data
livestock2 <- window(livestock, start = 1970, end = 2000)
fit1 <- ses(livestock2)
fit2 <- holt(livestock2)
fit3 <- holt(livestock2, damped = TRUE)

# Compare accuracy
accuracy(fit1, livestock)
accuracy(fit2, livestock)
accuracy(fit3, livestock)
```

## Seasonal Methods — Holt-Winters

Extends Holt to include a **seasonal component $s_t$**.

### Holt-Winters Additive

**Use when:** Seasonal amplitude is **constant** (does not grow with the level).

$$\hat{y}_{t+h|t} = \ell_t + h b_t + s_{t+h-m(k+1)}$$

$$\ell_t = \alpha(y_t - s_{t-m}) + (1-\alpha)(\ell_{t-1} + b_{t-1})$$

$$b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)b_{t-1}$$

$$s_t = \gamma(y_t - \ell_{t-1} - b_{t-1}) + (1-\gamma)s_{t-m}$$

Parameters: $0 \leq \alpha \leq 1$, $0 \leq \beta^* \leq 1$, $0 \leq \gamma \leq 1-\alpha$

$m$ = periodicity (e.g., 12 for monthly), $k = \lfloor(h-1)/m\rfloor$

### Holt-Winters Multiplicative

**Use when:** Seasonal amplitude **grows proportionally** with the level.

$$\hat{y}_{t+h|t} = (\ell_t + h b_t) s_{t+h-m(k+1)}$$

$$\ell_t = \alpha \frac{y_t}{s_{t-m}} + (1-\alpha)(\ell_{t-1} + b_{t-1})$$

$$s_t = \gamma \frac{y_t}{\ell_{t-1} + b_{t-1}} + (1-\gamma)s_{t-m}$$

### Holt-Winters Damped

Usually the **best performing method** for seasonal series. Adds the damping parameter $\phi$ to the multiplicative seasonal model.

### R code

```r
library(fpp2)

aust <- window(austourists, start = 2005)

# Fit additive and multiplicative Holt-Winters
fit1 <- hw(aust, seasonal = "additive")
fit2 <- hw(aust, seasonal = "multiplicative")

# Plot both
tmp <- cbind(Data = aust,
             "HW additive"       = fit1[["mean"]],
             "HW multiplicative" = fit2[["mean"]])
autoplot(tmp) + xlab("Year") + ylab("Hotel Stays (millions)")

# Damped multiplicative (usually best for seasonal TS)
fc <- hw(subset(hyndsight, end = length(hyndsight) - 35),
         damped = TRUE, seasonal = "multiplicative", h = 35)

# Extract estimated components
addstates  <- fit1$model$states[, 1:3]
multstates <- fit2$model$states[, 1:3]
colnames(addstates) <- colnames(multstates) <- c("level","slope","seasonality")
```

## Prediction Error Measurements

These metrics allow you to **compare competing models** on a test set.

| Metric | Formula | Notes |
|---|---|---|
| **MAE / MAD** | $\frac{\sum |Y_t - F_t|}{n}$ | Same units as data; easy to interpret |
| **MSE** | $\frac{\sum (Y_t - F_t)^2}{n-1}$ | Penalises large errors more heavily |
| **MAPE** | $\frac{1}{n}\sum\left|\frac{Y_t - F_t}{Y_t}\right| \times 100$ | Percentage; unit-free; allows comparison across series |
| **RMSE** | $\sqrt{MSE}$ | Same units as data; commonly reported |
| **MASE** | Scaled MAE | Useful when comparing across different scales |

Also used: **AIC**, **AICc**, **BIC** — information criteria for model selection (lower = better).

> **From the livestock example:** The Holt Linear method outperformed both SES and Holt Damped on the test set (lower RMSE, MAE, MAPE). Always evaluate on a **held-out test set**, not just training data.

## ETS Framework in R

**ETS** = **E**rror, **T**rend, **S**easonality. This unified framework encompasses all exponential smoothing methods.

Each component takes a type:

| Component | Types |
|---|---|
| Error | **A** (additive), **M** (multiplicative) |
| Trend | **N** (none), **A** (additive), **Ad** (additive damped) |
| Seasonality | **N** (none), **A** (additive), **M** (multiplicative) |

Examples: `ETS(A,N,N)` = SES; `ETS(A,A,A)` = Holt-Winters additive; `ETS(M,Ad,M)` = Holt-Winters damped multiplicative.

### R code

```r
library(fpp2)

aust <- window(austourists, start = 2005)

# Auto-select model by AICc
fit <- ets(aust)
summary(fit)
autoplot(fit)

# Force a specific model
ets(h02, model = "AAA", damped = FALSE)

# Train/test split workflow
train <- window(h02, end  = c(2004, 12))
test  <- window(h02, start = 2005)

fit1 <- ets(train)                                    # fit on train
fit2 <- ets(test, model = fit1, use.initial.values = TRUE)  # apply to test

accuracy(fit2)
accuracy(forecast(fit1, 10), test)

# Residual diagnostics
cbind('Residuals'      = residuals(fit),
      'Forecast errors' = residuals(fit, type = 'response')) %>%
  autoplot(facet = TRUE) + xlab("Year") + ylab("")
```

# Key Equations/Formulas Summary

**Decomposition:**
$$y_t = S_t + T_t + R_t \quad \text{(additive)} \qquad y_t = S_t \times T_t \times R_t \quad \text{(multiplicative)}$$

**Simple Moving Average:**
$$MA_p = \frac{\sum_{i=1}^{p} D_i}{p}$$

**SES forecast & smoothing:**
$$\hat{y}_{t+h|t} = \ell_t \qquad \ell_t = \alpha y_t + (1-\alpha)\ell_{t-1}$$

**Holt's linear trend:**
$$\hat{y}_{t+h|t} = \ell_t + hb_t \qquad b_t = \beta^*(\ell_t - \ell_{t-1}) + (1-\beta^*)b_{t-1}$$

**Optimisation criterion (SSE):**
$$\text{SSE} = \sum_{t=1}^{T}(y_t - \hat{y}_{t|t-1})^2$$

**Error metrics:**
$$\text{MAE} = \frac{\sum|Y_t - F_t|}{n} \qquad \text{MAPE} = \frac{1}{n}\sum\left|\frac{Y_t-F_t}{Y_t}\right|\times100$$

# Summary Points

- A time series is decomposed into **Trend + Seasonal + Cyclical + Residual**. The cyclical component is usually absorbed into the trend in practice.
- Choose **additive decomposition** for constant seasonality; **multiplicative** (or log-transform) for growing seasonality.
- **STL** is preferred over classical decomposition — it is robust to outliers and gives user control over smoothness.
- The **residuals** after decomposition must look **random** (no pattern, centred on zero, approximately normal). If they don't, the decomposition is incomplete.
- **SES** is for series with no trend and no seasonality. The parameter $\alpha$ controls how quickly the model "forgets" old data.
- **Holt's method** adds a trend component (linear or damped). The **damped version** is safer for long-horizon forecasts.
- **Holt-Winters** adds a seasonal component on top of Holt. The **damped multiplicative** version is often the best performer.
- Always evaluate models on a **test set** using error metrics (MAE, RMSE, MAPE). Minimise **AICc** for model selection within the ETS framework.
- `ets()` in R selects the best ETS model automatically using AICc — but use `ses()`, `holt()`, and `hw()` when you want direct control.
